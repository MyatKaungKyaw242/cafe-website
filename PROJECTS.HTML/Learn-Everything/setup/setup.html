Dev Setup Guide

Get your environment ready before writing a single line of code

🛠 Do this once — then you're set up for every project
💻

Phase 1 — Install VS Code

Your code editor — where you'll write everything

Step 1 of 5
1

Download VS Code

Free. The most popular code editor in the world.

Go to code.visualstudio.com and click the big download button. It will detect your OS (Windows/Mac/Linux) automatically.

  • Run the installer — click Next through all steps, keep all defaults
  • On the "Select Additional Tasks" screen: tick "Add to PATH" and "Open with Code" context menu — these are useful later
  • Click Install → Finish
  • Open VS Code — you should see the Welcome tab
✓ Done when: VS Code opens and you see the welcome screen.
2

Change the theme (optional but recommended)

A good theme reduces eye strain during long sessions

The default theme is fine, but many developers prefer a dark theme. Here's how to change it:

  • Press Ctrl+K then Ctrl+T — the theme picker opens
  • Popular choices: One Dark Pro, Dracula, Night Owl
  • To install a new theme: click the Extensions icon (4 squares) → search the theme name → Install → Set as theme
🧩

Phase 2 — Install Essential Extensions

Extensions add features to VS Code — install these before anything else

Step 2 of 5
1

How to install extensions

Takes 30 seconds each

  • Click the Extensions icon in the left sidebar (4 squares) — or press Ctrl+Shift+X
  • Type the extension name in the search box
  • Click the blue Install button
  • The extension activates immediately — no restart needed
2

The 6 extensions you must install

These make coding dramatically faster and less painful

Search and install each of these one by one:

Live Server by Ritwick Dey — auto-reloads your browser when you save. Essential for HTML/CSS/JS.
Prettier by Prettier — auto-formats your code on save. Keeps everything neat without thinking.
ESLint by Microsoft — highlights JavaScript errors and bad practices as you type.
Auto Rename Tag by Jun Han — rename an opening HTML tag and the closing tag renames too automatically.
ES7+ React Snippets by dsznajder — type "rafce" and it generates a full React component. Huge time saver.
Path Intellisense by Christian Kohler — autocompletes file paths when you type href or src. No more typos.
✓ Done when: all 6 show as "Installed" in the Extensions panel.
3

Configure Prettier — format on save

One setting that keeps your code clean automatically

Open VS Code settings and enable "Format on Save" so Prettier runs every time you hit Ctrl+S:

  • Press Ctrl+, to open Settings
  • Search for "format on save"
  • Tick the checkbox: Editor: Format On Save
  • Search for "default formatter"
  • Set it to Prettier - Code formatter
✓ Done when: saving a messy file auto-indents and formats it.
4

Learn the 5 most important shortcuts

These will save you hours every week

Ctrl + S          → Save file (do this constantly)
Ctrl + Z          → Undo
Ctrl + Shift + K  → Delete current line
Alt + Up/Down     → Move current line up or down
Ctrl + /          → Comment / uncomment a line
Ctrl + D          → Select next occurrence of selected word
Ctrl + `          → Open the integrated terminal
Ctrl + P          → Quick open any file by name
Ctrl + Shift + P  → Command palette (search any VS Code action)

Phase 3 — Install Node.js

Required for React (Vite), npm packages, and running JS outside the browser

Step 3 of 5
1

Download and install Node.js

You need this before you can use React or npm

Go to nodejs.org and download the LTS version (Long Term Support — the stable one). Do not download "Current" — it may have bugs.

  • Run the installer — click Next through all steps, keep all defaults
  • When it asks about "Tools for Native Modules" — you can skip this (uncheck it)
  • Click Install → Finish
  • Restart VS Code after installation
2

Verify Node.js installed correctly

Run these two commands in the terminal

Open the VS Code terminal with Ctrl+` and run these commands:

# Check Node.js version — should print something like v20.x.x
node --version

# Check npm version — should print something like 10.x.x
npm --version
  • If you see version numbers → Node.js is installed correctly ✓
  • If you see "not recognized" → restart VS Code and try again
  • If still not working → restart your computer, then try again
✓ Done when: both commands print version numbers without errors.

Phase 4 — HTML/CSS/JS Workflow (Live Server)

How to run your HTML projects and see changes instantly

Step 4 of 5
1

Create your first project folder

One folder = one project. Always.

  • Create a new folder somewhere on your computer — e.g. Documents/my-projects/practice-1
  • Open VS Code → File → Open Folder → select that folder
  • You should see the folder name in the Explorer panel on the left
2

Create your HTML file and launch Live Server

Your browser will auto-refresh every time you save

  • In VS Code Explorer panel → right-click → New File → name it index.html
  • Type ! and press Tab — VS Code generates a full HTML boilerplate
  • Add some content inside <body>
  • Right-click anywhere in the HTML file → "Open with Live Server"
  • Your browser opens at http://127.0.0.1:5500
  • Now every time you press Ctrl+S, the browser refreshes automatically
<!-- Your first index.html -->
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
  <title>My First Page</title>
  <link rel="stylesheet" href="style.css" />
</head>
<body>
  <h1>Hello World!</h1>
  <script src="main.js"></script>
</body>
</html>
✓ Done when: editing and saving the file updates the browser instantly without refreshing manually.
3

Recommended folder structure for HTML/CSS/JS projects

Keep files organized from day one

my-project/
├── index.html       ← main HTML file
├── css/
│   └── style.css    ← all your styles
├── js/
│   └── main.js      ← all your JavaScript
└── images/
    └── photo.jpg    ← images, icons

<!-- Link them with relative paths -->
<link rel="stylesheet" href="css/style.css" />
<script src="js/main.js"></script>

Phase 5 — React Workflow (Vite)

How to create and run a React project — the modern way

Step 5 of 5
1

Create a new React project with Vite

One command — Vite sets everything up for you

Open the VS Code terminal (Ctrl+`), navigate to where you want the project, then run:

# Create the project
npm create vite@latest my-app -- --template react

# Move into the project folder
cd my-app

# Install all dependencies
npm install

# Start the dev server
npm run dev
  • When it asks "Select a framework" → choose React
  • When it asks "Select a variant" → choose JavaScript (not TypeScript for now)
  • After npm run dev, open your browser at http://localhost:5173
  • You should see the Vite + React welcome page spinning
✓ Done when: you see the Vite + React logo at localhost:5173.
2

Understand the project structure

Know what each file does before you touch anything

my-app/
├── public/              ← static files (favicon, images) — copied as-is
├── src/                 ← YOUR code lives here
│   ├── App.jsx          ← root component — start editing here
│   ├── App.css          ← styles for App
│   ├── main.jsx         ← entry point (mounts App to the DOM — don't touch)
│   └── index.css        ← global styles
├── index.html           ← HTML shell (don't touch)
├── package.json         ← project info + dependencies list
└── vite.config.js       ← Vite configuration (don't touch for now)
  • Start in src/App.jsx — delete the template code and write your own
  • Create new components as src/components/MyComponent.jsx
  • Never edit main.jsx or index.html unless you know what you're doing
3

The 4 npm commands you'll use every day

Memorize these — you'll run them constantly

# Start the dev server (run this every time you open the project)
npm run dev

# Install a new package (e.g. react-router-dom)
npm install react-router-dom

# Build for production (creates optimized files in /dist folder)
npm run build

# Install all packages from package.json (after cloning someone else's project)
npm install
Important: Always run npm run dev from inside the project folder. If you see "npm: command not found", Node.js isn't installed or VS Code needs a restart.
4

Your first custom React component

Delete the template code and write this to confirm everything works

Open src/App.jsx, delete all the template code, and replace with this:

import { useState } from 'react'
import './App.css'

const App = () => {
  const [count, setCount] = useState(0)

  return (
    <div style={{ textAlign: 'center', padding: '40px' }}>
      <h1>My React App ⚛</h1>
      <p>Count: {count}</p>
      <button onClick={() => setCount(c => c + 1)}>
        Click me
      </button>
    </div>
  )
}

export default App
  • Save the file — the browser updates instantly (hot module replacement)
  • Click the button — the count should increment
  • If it works, your React setup is complete ✓
✓ Done when: clicking the button increments the count. Your environment is fully set up and ready.

Quick Reference — Daily Workflow

What you do every time you open VS Code to work on a project

Reference
✓

HTML/CSS/JS project — daily workflow

1. Open VS Code
2. File → Open Folder → select your project folder
3. Open index.html
4. Right-click → "Open with Live Server"
5. Browser opens at localhost:5500
6. Code → Ctrl+S to save → browser auto-refreshes
7. Keep DevTools (F12) open in the browser
✓

React project — daily workflow

1. Open VS Code
2. File → Open Folder → select your React project folder
3. Press Ctrl+` to open the terminal
4. Run: npm run dev
5. Browser opens at localhost:5173
6. Code in src/ → save → browser auto-updates (HMR)
7. Keep DevTools (F12) + Console open
8. When done: Ctrl+C in terminal to stop the dev server

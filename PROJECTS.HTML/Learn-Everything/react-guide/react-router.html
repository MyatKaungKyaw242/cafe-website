What is Routing?

Why React apps need a router

The Problem — React is a Single Page App

By default, a React app is a Single Page Application (SPA) — there is only one HTML file (index.html). There are no separate pages. If you navigate to /about, the browser would try to load a file that doesn't exist and show a 404 error.

React Router solves this by intercepting the URL and rendering different components based on what the URL says — without ever actually loading a new page from the server. The URL changes, but it's all still the same React app running in the background.

This gives you the feel of a multi-page website while keeping all the performance benefits of a single page app.

Real World Example — A Portfolio Website

A portfolio has 4 "pages": Home, Projects, About, Contact. Without React Router, you'd have to show/hide sections manually with state. With React Router, each URL maps to a real component — clean, professional, and shareable links.

  • / → renders the <Home /> component
  • /projects → renders the <Projects /> component
  • /about → renders the <About /> component
  • /projects/3 → renders <ProjectDetail /> for project with id 3

Quick Summary

  • React apps are Single Page Apps — one HTML file, no separate pages by default
  • React Router intercepts URL changes and renders the matching component
  • The page never actually reloads — it just swaps components
  • You get real URLs that are shareable and work with the browser back button
  • Library name: react-router-dom (for web apps)

Installation & Setup

Install React Router and wrap your app

Install the Package

React Router is not built into React — you install it as a separate package. Open your terminal inside your Vite React project and run:

# Install React Router (v6 — the current version)
npm install react-router-dom

Wrap Your App with BrowserRouter

The BrowserRouter component must wrap your entire application. It watches the browser's URL and tells all the routing components below it what URL is currently active. The best place to add it is in main.jsx — the entry point of your app.

// src/main.jsx
import React from 'react'
import ReactDOM from 'react-dom/client'
import { BrowserRouter } from 'react-router-dom'  // ← import
import App from './App.jsx'
import './index.css'

ReactDOM.createRoot(document.getElementById('root')).render(
  <BrowserRouter>     {/* ← wrap App with BrowserRouter */}
    <App />
  </BrowserRouter>
)

Project Structure with React Router

The standard way to organise a routed React app — one file per page in a pages/ folder, reusable UI in components/.

src/
├── pages/
│   ├── Home.jsx        ← "/" route
│   ├── About.jsx       ← "/about" route
│   ├── Projects.jsx    ← "/projects" route
│   ├── ProjectDetail.jsx ← "/projects/:id" route
│   └── NotFound.jsx    ← 404 catch-all route
├── components/
│   └── Navbar.jsx      ← shared across all pages
├── App.jsx             ← where Routes live
└── main.jsx            ← BrowserRouter lives here

Quick Summary

  • Install with: npm install react-router-dom
  • Wrap your app in <BrowserRouter> in main.jsx
  • Create a pages/ folder — one component per page
  • Routes are defined in App.jsx

Routes & Route

Map URLs to components

Defining Your Routes

<Routes> is the container for all your routes. Inside it, each <Route> maps a URL path to a component. When the URL matches, React Router renders that component in place of the <Routes> block.

Think of it like a switch statement for URLs — only the matching route renders.

// src/App.jsx
import { Routes, Route } from 'react-router-dom'
import Navbar    from './components/Navbar'
import Home      from './pages/Home'
import About     from './pages/About'
import Projects  from './pages/Projects'
import ProjectDetail from './pages/ProjectDetail'
import NotFound  from './pages/NotFound'

const App = () => {
  return (
    <div>
      <Navbar />  {/* always visible — outside Routes */}

      <Routes>
        {/* exact URL "/" → Home component */}
        <Route path="/"          element={<Home />} />

        {/* "/about" → About component */}
        <Route path="/about"     element={<About />} />

        {/* "/projects" → Projects component */}
        <Route path="/projects"  element={<Projects />} />

        {/* "/projects/1", "/projects/42" → ProjectDetail */}
        <Route path="/projects/:id" element={<ProjectDetail />} />

        {/* Catch-all — anything else → 404 page */}
        <Route path="*"          element={<NotFound />} />
      </Routes>
    </div>
  )
}

export default App

A Simple Page Component

Each page is just a normal React component. It doesn't need any special router code — React Router renders it when the URL matches.

// src/pages/Home.jsx
const Home = () => {
  return (
    <main>
      <h1>Welcome Home 👋</h1>
      <p>This is the home page.</p>
    </main>
  )
}

export default Home

// src/pages/About.jsx
const About = () => {
  return (
    <main>
      <h1>About Me</h1>
      <p>I'm learning React Router!</p>
    </main>
  )
}

export default About

❌ Common Mistakes

  • Putting <Navbar> inside <Routes> — it will only render when a route matches. Put it outside
  • Forgetting <Routes> wrapper — <Route> must always be inside <Routes>
  • Forgetting the catch-all path="*" — without it, bad URLs show nothing instead of a 404 page

Link & NavLink

Navigate between pages without page reloads

Never use <a href> for internal navigation

A regular HTML <a href="/about"> tag causes a full page reload — it throws away all your React state and re-fetches everything. React Router's <Link> and <NavLink> update the URL and swap components without reloading the page.

  • <Link> — basic navigation, like an anchor tag but no reload
  • <NavLink> — same as Link but automatically adds an active class when the current URL matches. Perfect for navbars.
// src/components/Navbar.jsx
import { Link, NavLink } from 'react-router-dom'
import './Navbar.css'

const Navbar = () => {
  return (
    <nav className="navbar">
      <Link to="/" className="logo">MyPortfolio</Link>

      <div className="nav-links">
        {/* NavLink automatically adds class="active" on the current page */}
        <NavLink to="/"        className={({ isActive }) => isActive ? "active" : ""}>Home</NavLink>
        <NavLink to="/about"   className={({ isActive }) => isActive ? "active" : ""}>About</NavLink>
        <NavLink to="/projects"className={({ isActive }) => isActive ? "active" : ""}>Projects</NavLink>
      </div>
    </nav>
  )
}

export default Navbar
/* Navbar.css */
.navbar {
  display: flex;
  align-items: center;
  justify-content: space-between;
  padding: 16px 40px;
  background: #1a1a2e;
  border-bottom: 1px solid #2a2a45;
}

.nav-links { display: flex; gap: 24px; }

.nav-links a {
  color: #888aaa;
  text-decoration: none;
  font-size: 0.9rem;
  font-weight: 600;
  transition: color 0.2s;
}

/* NavLink adds this class automatically on the active page */
.nav-links a.active,
.nav-links a:hover { color: white; }

❌ Common Mistakes

  • Using <a href="/about"> instead of <Link to="/about"> — causes full page reload and loses all React state
  • Using href prop on Link — it uses to, not href
  • Using <Link> for external URLs — use a regular <a href> with target="_blank" for external links

Dynamic Routes & useParams

URL parameters — /users/42, /products/shoes

What is a Dynamic Route?

A dynamic route has a variable segment in the URL — written with a colon prefix like :id. The same route component handles many different URLs — /products/1, /products/42, /products/shoes — and the variable part is available inside the component via the useParams hook.

This is how every product page, user profile page, and blog post page on the internet works.

// In App.jsx — define the dynamic route
<Route path="/projects/:id" element={<ProjectDetail />} />
//                    ^^^
//     ":id" is the parameter — matches any value
// src/pages/ProjectDetail.jsx
import { useParams, Link } from 'react-router-dom'
import { useState, useEffect } from 'react'

const projects = [
  { id: "1", title: "Todo App",       tech: "React, localStorage",    desc: "A full CRUD todo app." },
  { id: "2", title: "Weather App",    tech: "React, Fetch API",       desc: "Live weather by city." },
  { id: "3", title: "Portfolio Site", tech: "React, React Router",    desc: "This portfolio site!" },
]

const ProjectDetail = () => {
  const { id } = useParams()  // ← gets "1", "2", "3" etc. from the URL

  const project = projects.find(p => p.id === id)

  if (!project) {
    return (
      <div>
        <h2>Project not found!</h2>
        <Link to="/projects">← Back to projects</Link>
      </div>
    )
  }

  return (
    <main>
      <Link to="/projects">← All Projects</Link>
      <h1>{project.title}</h1>
      <p><strong>Tech:</strong> {project.tech}</p>
      <p>{project.desc}</p>
    </main>
  )
}

export default ProjectDetail
// src/pages/Projects.jsx — link to each project's detail page
import { Link } from 'react-router-dom'

const projects = [
  { id: "1", title: "Todo App" },
  { id: "2", title: "Weather App" },
  { id: "3", title: "Portfolio Site" },
]

const Projects = () => (
  <main>
    <h1>My Projects</h1>
    <ul>
      {projects.map(p => (
        <li key={p.id}>
          {/* Link to the dynamic route — /projects/1, /projects/2... */}
          <Link to={`/projects/${p.id}`}>{p.title}</Link>
        </li>
      ))}
    </ul>
  </main>
)

export default Projects

❌ Common Mistakes

  • useParams() always returns strings — if your ID is a number in the data, compare with p.id === id (string) not p.id === Number(id) unless you convert
  • Not handling the case where the param doesn't match anything — always check if the item was found and show a fallback
  • Using the wrong param name — :id in the route means const { id } = useParams(). They must match exactly

useNavigate

Navigate programmatically — after form submit, login, etc.

When to use useNavigate

<Link> is for navigation triggered by a click on a visible link. useNavigate is for navigation triggered by code logic — redirect after a form submission, send a user to the home page after login, go back after deleting an item.

import { useNavigate } from 'react-router-dom'

const ContactForm = () => {
  const navigate = useNavigate()
  const [name, setName]       = useState("")
  const [message, setMessage] = useState("")

  const handleSubmit = (e) => {
    e.preventDefault()
    // ... send the form data to an API

    // After submit → redirect to thank you page
    navigate("/thank-you")
  }

  return (
    <form onSubmit={handleSubmit}>
      <input value={name}    onChange={e => setName(e.target.value)}    placeholder="Name" />
      <textarea value={message} onChange={e => setMessage(e.target.value)} placeholder="Message" />
      <button type="submit">Send Message</button>
    </form>
  )
}

// Other common navigate patterns:
navigate("/")             // go to home
navigate(-1)              // go back (like browser back button)
navigate("/login")        // redirect to login
navigate(`/users/${id}`)  // go to a dynamic route
navigate("/home", { replace: true }) // redirect without adding to history

Quick Summary

  • const navigate = useNavigate() — call the hook to get the navigate function
  • navigate("/path") — go to a route
  • navigate(-1) — go back (like the browser back button)
  • navigate("/path", {"{ replace: true }"}) — redirect without adding to history
  • Use for: after form submit, after login/logout, after delete

Nested Routes & Outlet

Shared layouts with different page content

What are Nested Routes?

Nested routes let you share a layout (like a sidebar or tabs) across multiple pages, while only swapping the inner content. The parent route renders the shared layout. The <Outlet /> component marks where the child route's content should appear.

Real world example: a dashboard with a sidebar. The sidebar is always there, but the main area changes based on the URL.

// App.jsx — Dashboard has nested routes
<Routes>
  <Route path="/"         element={<Home />} />

  {/* Dashboard layout wraps these child routes */}
  <Route path="/dashboard" element={<DashboardLayout />}>
    <Route index           element={<DashboardHome />} />  {/* /dashboard */}
    <Route path="profile"  element={<Profile />} />        {/* /dashboard/profile */}
    <Route path="settings" element={<Settings />} />       {/* /dashboard/settings */}
  </Route>

  <Route path="*" element={<NotFound />} />
</Routes>
// src/pages/DashboardLayout.jsx
import { NavLink, Outlet } from 'react-router-dom'

const DashboardLayout = () => (
  <div style={{ display: "grid", gridTemplateColumns: "200px 1fr" }}>

    {/* This sidebar shows on ALL dashboard pages */}
    <aside>
      <NavLink to="/dashboard"          end>Overview</NavLink>
      <NavLink to="/dashboard/profile"     >Profile</NavLink>
      <NavLink to="/dashboard/settings"    >Settings</NavLink>
    </aside>

    {/* Outlet = where the child route renders */}
    <main>
      <Outlet />
    </main>

  </div>
)

export default DashboardLayout

// Note: use "end" on the index NavLink so it only
// shows as active on the exact /dashboard URL

Quick Summary

  • Nest <Route> components inside a parent <Route> to share a layout
  • The parent component renders <Outlet /> where the child page content goes
  • index route = the default child when no sub-path is specified
  • Use for: dashboards with sidebars, tabs, settings pages with sub-sections
  • Add end prop to NavLink for the index route so it only highlights exactly

404 Page & Full Setup Summary

Handle unknown routes + everything in one place

The 404 Catch-All Route

Any URL that doesn't match any of your routes should show a friendly 404 page instead of a blank screen. Use path="*" as the last route — it catches everything that didn't match above it.

// src/pages/NotFound.jsx
import { Link } from 'react-router-dom'

const NotFound = () => (
  <div style={{ textAlign: "center", padding: "80px 20px" }}>
    <h1 style={{ fontSize: "4rem" }}>404</h1>
    <p style={{ color: "#888" }}>Oops — this page doesn't exist.</p>
    <Link to="/">← Go Home</Link>
  </div>
)

export default NotFound

// In App.jsx — must be the LAST route
<Route path="*" element={<NotFound />} />

Complete App.jsx — Everything Together

Here's a full, production-ready routing setup you can use as a template for any project:

// src/App.jsx — complete routing template
import { Routes, Route } from 'react-router-dom'

// Layout components (always visible)
import Navbar from './components/Navbar'
import Footer from './components/Footer'

// Page components
import Home           from './pages/Home'
import About          from './pages/About'
import Projects       from './pages/Projects'
import ProjectDetail  from './pages/ProjectDetail'
import Contact        from './pages/Contact'
import NotFound       from './pages/NotFound'

const App = () => (
  <>
    {/* Navbar always renders — it's outside Routes */}
    <Navbar />

    {/* Only the matching route renders here */}
    <Routes>
      <Route path="/"            element={<Home />}          />
      <Route path="/about"       element={<About />}         />
      <Route path="/projects"    element={<Projects />}      />
      <Route path="/projects/:id"element={<ProjectDetail />} />
      <Route path="/contact"     element={<Contact />}       />
      <Route path="*"            element={<NotFound />}      />
    </Routes>

    {/* Footer always renders */}
    <Footer />
  </>
)

export default App

❌ Common Mistakes

  • Forgetting to wrap the app with <BrowserRouter> in main.jsx — you'll get "useRoutes() may be used only in the context of a Router" error
  • Using <a href> instead of <Link to> — causes full page reload
  • Putting the path="*" catch-all before other routes — it will match everything. It must be last
  • Using navigate() instead of <Link> for regular nav items — Link is for visible links, navigate is for code-triggered redirects
  • Forgetting <Outlet /> in the parent layout for nested routes — the child pages render nowhere

Full React Router Cheatsheet

  • npm install react-router-dom — install the package
  • <BrowserRouter> in main.jsx — wraps the whole app
  • <Routes> <Route path="/" element={<Home />} /> </Routes> — define routes in App.jsx
  • <Link to="/about"> — navigate without page reload
  • <NavLink> — Link that adds active class on current page
  • const {"{ id }"} = useParams() — read URL parameters (:id)
  • const navigate = useNavigate() then navigate("/path") — programmatic navigation
  • <Route path="*"> — 404 catch-all, always last
  • <Outlet /> — where child routes render in nested routing

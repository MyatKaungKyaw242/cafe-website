Every hook, JSX rule, and pattern you need — one page quick reference

JSX Rules

One Root Element

Every component must return one root element. Wrap with a div or Fragment.

// BAD
return (
  <h1>Title</h1>
  <p>Text</p>
);

// GOOD — Fragment
return (
  <>
    <h1>Title</h1>
    <p>Text</p>
  </>
);

className not class

JSX uses className for CSS classes. for → htmlFor.

// HTML
<div class="card"></div>
<label for="email"></label>

// JSX
<div className="card"></div>
<label htmlFor="email"></label>

Self-closing Tags

All tags must be closed. Self-close tags with no children.

// BAD
<img src="photo.jpg">
<input type="text">
<br>

// GOOD
<img src="photo.jpg" />
<input type="text" />
<br />

JS Expressions in JSX

Use {} to embed any JS expression — variables, functions, ternaries.

const name = "Myat";
const age = 20;

return (
  <div>
    <h1>{name}</h1>
    <p>Age: {age}</p>
    <p>{age >= 18 ? "Adult" : "Minor"}</p>
    <p>{name.toUpperCase()}</p>
  </div>
);

Inline Styles

Pass a JS object. Properties use camelCase, values are strings or numbers.

// Double {{ }} — outer is JSX, inner is object
<div style={{ color: "red", fontSize: 16 }}>
  Hello
</div>

// Or use a variable
const style = {
  background: "#1a1a2e",
  padding: "16px",
};
<div style={style}>Card</div>

Comments in JSX

JSX comments go inside {}. Regular // comments work outside JSX.

return (
  <div>
    {/* This is a JSX comment */}
    <h1>Title</h1>
    {/* TODO: add button here */}
  </div>
);
Components & Props

Function Component

A component is a function that returns JSX. Name must be capitalized.

// Arrow function (common)
const Button = () => {
  return <button>Click Me</button>;
};

// Regular function (also fine)
function Button() {
  return <button>Click Me</button>;
}

export default Button;

Passing Props

Props are like HTML attributes — pass any value including functions.

// Parent passes props
<Card
  title="Hello"
  count={42}
  isActive={true}
  onClick={handleClick}
/>

// Child receives props
const Card = ({ title, count, isActive, onClick }) => (
  <div onClick={onClick}>
    <h2>{title}</h2>
    <p>{count}</p>
  </div>
);

Default Props

Set fallback values directly in the destructuring parameter.

const Button = ({
  label = "Click Me",
  color = "blue",
  disabled = false
}) => (
  <button
    style={{ background: color }}
    disabled={disabled}
  >
    {label}
  </button>
);

// Works without any props
<Button />

children Prop

children is whatever you put between opening and closing tags.

const Card = ({ children, title }) => (
  <div className="card">
    <h2>{title}</h2>
    <div className="body">
      {children}
    </div>
  </div>
);

// Use it like an HTML tag
<Card title="My Card">
  <p>Any content here</p>
  <button>Action</button>
</Card>

Spread Props

Pass all properties of an object as props at once using spread.

const user = {
  name: "Myat",
  age: 20,
  role: "admin"
};

// Spread passes all 3 props at once
<UserCard {...user} />

// Same as:
<UserCard
  name={user.name}
  age={user.age}
  role={user.role}
/>

Props are Read-Only

Never modify props. If you need to change a value, use state instead.

// BAD — never mutate props
const Counter = ({ count }) => {
  count = count + 1; // ❌ wrong!
  return <p>{count}</p>;
};

// GOOD — copy to state if you need to change it
const Counter = ({ initialCount }) => {
  const [count, setCount] = useState(initialCount);
  return <p>{count}</p>;
};
Hooks Quick Reference

useState

Declare reactive state. Re-renders the component when state changes.

const [value, setValue] = useState(initialValue);

// Examples
const [count,   setCount]   = useState(0);
const [name,    setName]    = useState("");
const [isOpen,  setIsOpen]  = useState(false);
const [items,   setItems]   = useState([]);
const [user,    setUser]    = useState(null);

// Update
setCount(5);                         // set directly
setCount(prev => prev + 1);          // use previous value (safer)
setItems(prev => [...prev, newItem]);// add to array — never mutate!
setUser(prev => ({ ...prev, name: "New" })); // update object field

useEffect

Run side effects after render — fetch data, set up subscriptions, update the document title.

// Runs after EVERY render
useEffect(() => { console.log("rendered"); });

// Runs ONCE on mount (empty deps array)
useEffect(() => {
  fetchData();
}, []);

// Runs when 'userId' changes
useEffect(() => {
  fetchUser(userId);
}, [userId]);

// Cleanup — runs before next effect + on unmount
useEffect(() => {
  const timer = setInterval(tick, 1000);
  return () => clearInterval(timer);  // cleanup
}, []);

useRef

Access DOM elements directly, or store values that don't trigger re-renders.

const myRef = useRef(initialValue);

// DOM access — attach with ref prop
const inputRef = useRef(null);
<input ref={inputRef} />
inputRef.current.focus(); // access the real DOM element

// Store a value (no re-render when changed)
const timerRef = useRef(null);
timerRef.current = setInterval(tick, 1000);
clearInterval(timerRef.current);

// Track previous value
const prevValueRef = useRef(value);
useEffect(() => { prevValueRef.current = value; }); // update after render

useContext

Read a value from a Context without prop drilling. Pair with createContext.

// 1. Create context
const ThemeCtx = createContext("light");

// 2. Provide it high up
<ThemeCtx.Provider value="dark">
  <App />
</ThemeCtx.Provider>

// 3. Consume anywhere below
const theme = useContext(ThemeCtx);
// theme === "dark"

useReducer

Like useState but for complex state with multiple update types.

const reducer = (state, action) => {
  switch (action.type) {
    case "inc": return { count: state.count + 1 };
    case "dec": return { count: state.count - 1 };
    case "reset": return { count: 0 };
    default: return state;
  }
};

const [state, dispatch] = useReducer(reducer, { count: 0 });
dispatch({ type: "inc" });

useMemo & useCallback

Memoize expensive values or functions to avoid unnecessary recalculations.

// useMemo — cache a computed value
const total = useMemo(
  () => items.reduce((s, i) => s + i.price, 0),
  [items]  // recalculate only when items changes
);

// useCallback — cache a function
const handleClick = useCallback(() => {
  doSomething(id);
}, [id]);  // new function only when id changes
Events

Common Events

React events use camelCase. Pass a function reference, not a call.

// Click
<button onClick={handleClick}>

// Input change
<input onChange={handleChange}>

// Form submit
<form onSubmit={handleSubmit}>

// Keyboard
<input onKeyDown={handleKey}>

// Mouse
<div onMouseEnter={handleHover}>

// Focus
<input onFocus={handleFocus}
       onBlur={handleBlur}>

Event Object

React passes a synthetic event object. Access value, key, type, target.

const handleChange = (e) => {
  console.log(e.target.value);  // input value
  console.log(e.target.name);   // input name attr
  console.log(e.target.checked); // checkbox
};

const handleKey = (e) => {
  if (e.key === "Enter") submit();
  if (e.key === "Escape") close();
};

const handleSubmit = (e) => {
  e.preventDefault(); // stop page reload!
};

Passing Arguments

Wrap in an arrow function to pass extra arguments to your handler.

// Pass an ID to the handler
<button onClick={() => handleDelete(item.id)}>
  Delete
</button>

// Pass both event and data
<input
  onChange={(e) => handleChange(e, field)}
/>

// Inside the handler
const handleDelete = (id) => {
  setItems(prev => prev.filter(i => i.id !== id));
};

Preventing Defaults

Stop default browser behavior and stop events from bubbling up.

// Stop form page reload
const onSubmit = (e) => {
  e.preventDefault();
};

// Stop event bubbling to parent
const onCardClick = (e) => {
  e.stopPropagation();
};

// Stop link navigation
const onLink = (e) => {
  e.preventDefault();
  // do something custom
};
Lists & Conditional Rendering

Render a List

Use .map() to turn an array into JSX. Always add a unique key.

const fruits = ["Apple", "Mango", "Banana"];

return (
  <ul>
    {fruits.map((fruit, i) => (
      <li key={i}>{fruit}</li>
    ))}
  </ul>
);

// With objects — use the id as key
{users.map(user => (
  <UserCard key={user.id} user={user} />
))}

Filter + Map

Chain .filter() before .map() to show only matching items.

const [search, setSearch] = useState("");

const visible = items
  .filter(item =>
    item.name.toLowerCase()
      .includes(search.toLowerCase())
  )
  .map(item => (
    <Item key={item.id} {...item} />
  ));

return <ul>{visible}</ul>;

Ternary — Show A or B

Use condition ? A : B to switch between two outputs.

{isLoggedIn
  ? <Dashboard />
  : <LoginPage />
}

{isLoading
  ? <Spinner />
  : <Content data={data} />
}

// Inline text
<button>
  {isOpen ? "Close" : "Open"}
</button>

&& — Show or Nothing

Use condition && element to show something or nothing.

{isAdmin && <AdminPanel />}

{error && <p className="err">{error}</p>}

{/* ⚠️ Watch out for 0 */}
{count && <Badge />}    // renders "0" when empty!
{count > 0 && <Badge />} // renders nothing ✅

{/* Conditional class */}
<div className={`btn ${isActive ? "active" : ""}`}>

Early Return

Return early from the component for loading, error, or empty states.

const UserCard = ({ user }) => {
  if (!user)           return null;
  if (user.loading)    return <Spinner />;
  if (user.error)      return <p>Error!</p>;
  if (!user.name)      return <p>No name</p>;

  // Happy path at the bottom
  return (
    <div>
      <h2>{user.name}</h2>
    </div>
  );
};
Forms & Controlled Inputs

Controlled Input Pattern

Wire value to state and onChange to a setter. React is the single source of truth.

const [name, setName] = useState("");

<input
  value={name}
  onChange={(e) => setName(e.target.value)}
  placeholder="Your name"
/>

// Text area — same pattern
<textarea value={bio} onChange={(e) => setBio(e.target.value)} />

// Select dropdown
<select value={role} onChange={(e) => setRole(e.target.value)}>
  <option value="user">User</option>
  <option value="admin">Admin</option>
</select>

// Checkbox — use checked, not value
<input type="checkbox" checked={agreed} onChange={(e) => setAgreed(e.target.checked)} />

Multi-field Form (Generic Handler)

One handler for all fields using the name attribute as the key.

const [form, setForm] = useState({ name: "", email: "", password: "" });

const handleChange = (e) => {
  const { name, value } = e.target;
  setForm(prev => ({ ...prev, [name]: value }));
};

const handleSubmit = (e) => {
  e.preventDefault();      // always prevent page reload
  console.log(form);       // access all values here
};

return (
  <form onSubmit={handleSubmit}>
    <input name="name"     value={form.name}     onChange={handleChange} />
    <input name="email"    value={form.email}    onChange={handleChange} />
    <input name="password" value={form.password} onChange={handleChange} type="password" />
    <button type="submit">Submit</button>
  </form>
);
State Patterns

Update Array State

Never mutate. Always return a new array.

// Add item
setItems(prev => [...prev, newItem]);

// Remove item
setItems(prev => prev.filter(i => i.id !== id));

// Update one item
setItems(prev => prev.map(i =>
  i.id === id ? { ...i, done: true } : i
));

// Sort
setItems(prev =>
  [...prev].sort((a, b) => a.name.localeCompare(b.name))
);

Update Object State

Spread the old object, then overwrite only the changed field.

const [user, setUser] = useState({
  name: "Myat", age: 20, role: "user"
});

// Update one field
setUser(prev => ({ ...prev, age: 21 }));

// Update nested object
setUser(prev => ({
  ...prev,
  address: { ...prev.address, city: "Yangon" }
}));

// Reset to initial
setUser({ name: "", age: 0, role: "user" });

Toggle Boolean State

Use the functional update form so you always toggle the current value.

const [isOpen, setIsOpen] = useState(false);
const [isDark, setIsDark] = useState(false);

// Toggle
setIsOpen(prev => !prev);
setIsDark(prev => !prev);

// Button
<button onClick={() => setIsOpen(o => !o)}>
  {isOpen ? "Close" : "Open"}
</button>

// Don't use setIsOpen(!isOpen) — stale closure risk
// Always use the functional form for toggles

Lifting State Up

Move shared state to the closest common parent. Pass down data + callbacks.

// Parent owns the state
const App = () => {
  const [query, setQuery] = useState("");
  return (
    <>
      <SearchBar onSearch={setQuery} />
      <ResultList filter={query} />
    </>
  );
};

// Child reads or changes it via props
const SearchBar = ({ onSearch }) => (
  <input onChange={e => onSearch(e.target.value)} />
);
Fetching Data

Complete Fetch Pattern

Always track 3 states: loading, data, error. Fetch inside useEffect with empty deps array.

const [data,    setData]    = useState(null);
const [loading, setLoading] = useState(true);
const [error,   setError]   = useState(null);

useEffect(() => {
  const fetchData = async () => {
    try {
      setLoading(true);
      setError(null);
      const res = await fetch("https://api.example.com/data");
      if (!res.ok) throw new Error(`HTTP ${res.status}`);
      const json = await res.json();
      setData(json);
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };
  fetchData();
}, []);  // ← empty array = fetch once on mount

if (loading) return <Spinner />;
if (error)   return <p>Error: {error}</p>;
return <div>{/* render data */}</div>;

useFetch Custom Hook

Extract the fetch pattern into a reusable hook. Use it in any component.

// hooks/useFetch.js
const useFetch = (url) => {
  const [data,    setData]    = useState(null);
  const [loading, setLoading] = useState(true);
  const [error,   setError]   = useState(null);

  useEffect(() => {
    const run = async () => {
      try {
        setLoading(true);
        const res = await fetch(url);
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        setData(await res.json());
      } catch (e) {
        setError(e.message);
      } finally {
        setLoading(false);
      }
    };
    run();
  }, [url]);

  return { data, loading, error };
};

// In any component:
const { data, loading, error } = useFetch("/api/users");
const { data: posts }          = useFetch("/api/posts");
Common Patterns

Component File Structure

Standard layout for a component file — imports, component, export.

// 1. Imports
import { useState, useEffect } from 'react';
import './Button.css';

// 2. Component (arrow function)
const Button = ({ label, onClick, disabled = false }) => {
  const [clicked, setClicked] = useState(false);

  const handleClick = () => {
    setClicked(true);
    onClick?.();
  };

  return (
    <button onClick={handleClick} disabled={disabled}>
      {label}
    </button>
  );
};

// 3. Export
export default Button;

# Create a new project
npm create vite@latest my-app -- --template react
cd my-app
npm install
npm run dev

# Vite runs at http://localhost:5173

# Build for production
npm run build

# Key files:
# src/App.jsx       — your main component
# src/main.jsx      — mounts React to the DOM
# index.html        — HTML shell

Key Rules to Remember

The most important React rules — violating these causes bugs.

// 1. Never mutate state
arr.push(x);         // ❌
setArr([...arr, x]); // ✅

// 2. Hooks only at top level
if (x) { useState() } // ❌ never in conditions

// 3. Keys must be unique and stable
key={Math.random()} // ❌ creates new key every render
key={item.id}       // ✅

// 4. Functional updates for derived state
setCount(count + 1);       // ❌ stale closure risk
setCount(prev => prev + 1);// ✅

// 5. Always e.preventDefault() on submit
<form onSubmit={handleSubmit}> // and call e.preventDefault() inside

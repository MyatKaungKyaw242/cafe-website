01

Variables & Data Types

Pro Tip

Declare variables at the top of their scope and group const declarations together before let. This makes it instantly clear what changes and what doesn't — the most important signal for any reader of your code.

Trick

Use object destructuring as a named parameter pattern for functions with many options. Instead of fn(true, false, 20, "blue") (confusing), use fn({ isActive: true, size: 20, color: "blue" }) — readable at the call site.

// Confusing
createCard(true, false, 20, "blue");

// Clear
createCard({ isActive: true, hasBorder: false, size: 20, color: "blue" });
Trick

Use console.table() instead of console.log() for arrays of objects — displays them as a formatted table in DevTools. Massive time saver when debugging API responses.

const users = [{ name: "Myat", age: 19 }, { name: "Bob", age: 25 }];
console.table(users); // Shows a clean table — not messy objects
02

Operators & Coercion

Pro Tip

Always use === — never ==. The one exception: checking for both null and undefined at once: value == null catches both. This is the only time == is intentionally useful.

// Catches BOTH null and undefined
if (value == null) { /* handles both cases */ }

// Same as writing:
if (value === null || value === undefined) { }
Trick

Use the double negation trick !! to convert any value to its boolean equivalent. Useful for checking if something is "truthy" and you need a real boolean stored in a variable.

const username = "Myat";
const hasUsername = !!username; // true

const empty = "";
const hasEmpty = !!empty;       // false

// Same as Boolean(value) but shorter
03

Functions

Pro Tip

Write functions that do one thing and return a value. A function that both modifies DOM AND fetches data AND updates state is impossible to test and reuse. Single-responsibility functions are the mark of experienced developers.

Trick

Use early returns to avoid deeply nested if/else. This pattern is called a "guard clause" — handle the error/edge case first and return, leaving the happy path clean and at the lowest indent level.

// Bad — nested
const processUser = (user) => {
  if (user) {
    if (user.isActive) {
      if (user.age >= 18) {
        return doWork(user);
      }
    }
  }
};

// Good — early returns (guard clauses)
const processUser = (user) => {
  if (!user)          return null;
  if (!user.isActive) return null;
  if (user.age < 18)  return null;
  return doWork(user); // clean happy path
};
04

Control Flow

Pro Tip

Prefer lookup objects over long if/else or switch chains. When you have 5+ conditions mapping one value to another, a plain object is faster to write, read, and extend.

// Bad — long switch
const getColor = (status) => {
  switch(status) {
    case "success": return "green";
    case "error":   return "red";
    case "warning": return "orange";
    case "info":    return "blue";
    default:        return "grey";
  }
};

// Good — lookup object
const STATUS_COLORS = {
  success: "green",
  error:   "red",
  warning: "orange",
  info:    "blue"
};
const getColor = (status) => STATUS_COLORS[status] ?? "grey";
Trick

Use the optional chaining + nullish coalescing combo for safe nested access with a fallback in one line. This replaces 3-4 lines of defensive checks.

// 4 lines without
let city = "Unknown";
if (user && user.address && user.address.city) {
  city = user.address.city;
}

// 1 line with
const city = user?.address?.city ?? "Unknown";
05

Loops

Pro Tip

Prefer array methods over manual loops for data transformation. map, filter, reduce are more readable and express WHAT you want, not HOW to get it. Use for...of when you genuinely need to break early or use await inside the loop.

Trick

When you need to await inside a loop, forEach will NOT wait — use for...of instead. This is one of the most common async bugs beginners make.

const ids = [1, 2, 3, 4, 5];

// BROKEN — forEach doesn't await
ids.forEach(async (id) => {
  const data = await fetchItem(id); // fires all at once!
});

// CORRECT — for...of awaits each one
for (const id of ids) {
  const data = await fetchItem(id); // waits for each
}

// OR — run all in parallel (faster if order doesn't matter)
const results = await Promise.all(ids.map(id => fetchItem(id)));
06

Arrays & Methods

Pro Tip

Chain array methods for clean, readable data pipelines. Each method returns a new array, so you can chain them directly. This is the functional programming style used in every React codebase.

const products = [
  { name: "Shoes",  price: 80,  inStock: true,  category: "footwear" },
  { name: "Shirt",  price: 25,  inStock: true,  category: "clothing" },
  { name: "Bag",    price: 120, inStock: false, category: "accessories" },
  { name: "Hat",    price: 30,  inStock: true,  category: "clothing" },
];

// One pipeline: filter → sort → map
const result = products
  .filter(p => p.inStock)
  .sort((a, b) => a.price - b.price)
  .map(p => `${p.name}: $${p.price}`);

// ["Shirt: $25", "Hat: $30", "Shoes: $80"]
Trick

Use spread + Set to remove duplicate values from an array in one line.

const nums = [1, 2, 2, 3, 3, 3, 4];
const unique = [...new Set(nums)]; // [1, 2, 3, 4]

const tags = ["css", "js", "css", "html", "js"];
const uniqueTags = [...new Set(tags)]; // ["css", "js", "html"]
07

Objects & Destructuring

Pro Tip

Use Object.entries() to loop over key-value pairs cleanly — much better than for...in with a bracket access call.

const scores = { Alice: 95, Bob: 82, Charlie: 77 };

// Clean with entries + destructuring
Object.entries(scores).forEach(([name, score]) => {
  console.log(`${name}: ${score}`);
});

// Also useful with map to transform to array
const list = Object.entries(scores).map(([name, score]) => ({
  name,
  score,
  grade: score >= 90 ? "A" : "B"
}));
Trick

Use computed property names to dynamically build objects — extremely useful when handling form fields where the field name is stored in a variable.

// Dynamic form update
const updateField = (state, fieldName, value) => ({
  ...state,
  [fieldName]: value  // fieldName is a variable, not a literal key
});

const form = { name: "", email: "", age: "" };
const updated = updateField(form, "email", "myat@email.com");
// { name: "", email: "myat@email.com", age: "" }
08

DOM Manipulation

Pro Tip

When rendering a list of items, build the entire HTML string first, then set innerHTML once. Setting innerHTML inside a loop causes a browser re-render on every iteration — building first is dramatically faster.

// Slow — triggers reflow for every item
items.forEach(item => {
  container.innerHTML += `<div>${item}</div>`; // BAD!
});

// Fast — one innerHTML assignment
container.innerHTML = items
  .map(item => `<div class="item">${item}</div>`)
  .join("");
Trick

Use data attributes (data-*) to store IDs or state on DOM elements. Then you can retrieve them in event handlers without needing to maintain a separate mapping.

// Render with data-id
users.forEach(user => {
  list.innerHTML += `
    <li data-id="${user.id}" data-role="${user.role}">
      ${user.name}
      <button class="delete-btn">Delete</button>
    </li>`;
});

// In event handler — read back the id
list.addEventListener("click", (e) => {
  if (e.target.classList.contains("delete-btn")) {
    const id = e.target.closest("li").dataset.id;
    deleteUser(id); // no mapping needed!
  }
});
09

Events

Pro Tip

Use debouncing for search inputs and resize handlers — don't fire the function on every single keystroke or pixel scrolled. Wait until the user pauses. This prevents unnecessary API calls and improves performance.

// Simple debounce function
const debounce = (fn, delay) => {
  let timer;
  return (...args) => {
    clearTimeout(timer);
    timer = setTimeout(() => fn(...args), delay);
  };
};

const searchInput = document.querySelector("#search");
const handleSearch = debounce((e) => {
  fetchResults(e.target.value); // only fires 400ms after user stops typing
}, 400);

searchInput.addEventListener("input", handleSearch);
Trick

Use { once: true } as the third argument to addEventListener when you only want the event to fire once — like an intro animation or a one-time confirmation. Cleaner than removing the listener manually.

btn.addEventListener("click", handleFirstClick, { once: true });
// Auto-removes itself after the first click
10

ES6+ Syntax

Pro Tip

Use tagged template literals for building safe HTML strings — and make it a habit to sanitize user input before putting it in innerHTML to prevent XSS attacks.

// Never trust user input in innerHTML
const safeText = (str) => str
  .replace(/&/g, "&amp;")
  .replace(/</g, "&lt;")
  .replace(/>/g, "&gt;");

// Then use safely
card.innerHTML = `<p>${safeText(userInput)}</p>`;
Trick

Use array destructuring to swap variables without a temp variable — and to cleanly extract multiple return values from a function.

// Swap without temp variable
let a = 1, b = 2;
[a, b] = [b, a]; // a=2, b=1

// Return multiple values cleanly
const getMinMax = (arr) => [Math.min(...arr), Math.max(...arr)];
const [min, max] = getMinMax([3, 1, 7, 2, 9]);
console.log(min, max); // 1, 9

Fetch API & Async/Await

Pro Tip

Use Promise.all() to run multiple fetch calls in parallel instead of sequentially. If you need 3 API calls that don't depend on each other, running them in parallel can be 3x faster.

// Sequential — slow (waits for each)
const user    = await fetchUser(id);
const posts   = await fetchPosts(id);
const comments = await fetchComments(id);

// Parallel — fast (all run at the same time)
const [user, posts, comments] = await Promise.all([
  fetchUser(id),
  fetchPosts(id),
  fetchComments(id)
]);
Trick

Create a reusable fetch wrapper so you don't repeat try/catch + response.ok check in every single function. One place to handle all API errors.

// api.js
const apiFetch = async (url, options = {}) => {
  const response = await fetch(url, options);
  if (!response.ok) throw new Error(`API error: ${response.status}`);
  return response.json();
};

// Usage — clean, no repeated error boilerplate
const users = await apiFetch("/api/users");
const post  = await apiFetch("/api/posts/1");
const newPost = await apiFetch("/api/posts", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify(data)
});

Error Handling

Pro Tip

Always log the full error object, not just the message. console.error(error) gives you the stack trace — the exact file and line number where it broke. error.message alone tells you what but not where.

try {
  await fetchData();
} catch (error) {
  console.error(error);         // Full stack trace
  console.error(error.message); // Just the message — less useful for debugging
  showUserError("Something went wrong. Try again."); // User-facing
}
Trick

Use a global error handler to catch any uncaught promise rejections or errors — a safety net for your whole app.

// Catch uncaught promise rejections
window.addEventListener("unhandledrejection", (event) => {
  console.error("Unhandled promise rejection:", event.reason);
  // Show a generic error notification to the user
});

// Catch any uncaught sync errors
window.addEventListener("error", (event) => {
  console.error("Global error:", event.error);
});

Scope & Closures

Pro Tip

Use closures to create factory functions — functions that create specialized versions of themselves. This avoids repeating configuration parameters every time you call a function.

// Factory function — creates a multiplier
const makeMultiplier = (factor) => (n) => n * factor;

const double = makeMultiplier(2);
const triple = makeMultiplier(3);
const tenX   = makeMultiplier(10);

double(5);  // 10
triple(5);  // 15
tenX(5);    // 50
Trick

Use IIFE (Immediately Invoked Function Expression) to run code in isolation without polluting the global scope — useful for init code that runs once on page load.

// IIFE — runs immediately, variables stay private
(() => {
  const config = loadConfig();
  initApp(config);
  bindEvents();
  console.log("App initialized");
})(); // <-- the () at the end invokes it immediately

Local Storage

Pro Tip

Wrap all localStorage calls in a storage helper module with try/catch. localStorage can fail in private browsing mode or when storage is full — unhandled errors crash your whole app silently.

// storage.js — safe wrapper
export const storage = {
  get: (key, fallback = null) => {
    try {
      const item = localStorage.getItem(key);
      return item ? JSON.parse(item) : fallback;
    } catch { return fallback; }
  },
  set: (key, value) => {
    try {
      localStorage.setItem(key, JSON.stringify(value));
    } catch { console.warn("Storage full or unavailable"); }
  },
  remove: (key) => localStorage.removeItem(key)
};
Trick

Add a version key to your localStorage data. When you change your data structure, increment the version and clear old data — prevents crashes from stale saved formats.

const DATA_VERSION = "v2";
const saved = storage.get("app-data");

if (saved?.version !== DATA_VERSION) {
  storage.remove("app-data"); // wipe old format
  initFreshData();
} else {
  loadSavedData(saved);
}
15

Modules

Pro Tip

Follow the barrel export pattern — create an index.js in each folder that re-exports everything from that folder's files. Then import from the folder path, not the individual file. Cleaner imports that don't break when you rename files.

// utils/index.js — barrel file
export { formatPrice } from "./formatPrice.js";
export { capitalize }  from "./capitalize.js";
export { debounce }    from "./debounce.js";

// In main.js — import from folder, not individual file
import { formatPrice, capitalize, debounce } from "./utils/index.js";
// or just: import { formatPrice } from "./utils";
Trick

Use dynamic imports to only load heavy modules when you actually need them — keeps initial page load fast.

// Load a chart library only when user opens the chart tab
const chartBtn = document.querySelector("#show-chart");
chartBtn.addEventListener("click", async () => {
  const { renderChart } = await import("./chart.js"); // loads now
  renderChart(data);
});

01

grid-template-areas

Pro Tip

Use a . (dot) to create empty cells. Perfect for layouts where you want to skip a column without adding a dummy element.

grid-template-areas:
  "header header header"
  ".      main   sidebar"
  "footer footer footer";
Pro Tip

Name your areas by purpose, not by tag. Use hero not div1. Future you will thank you.

Pro Tip

You can change the entire page layout at a breakpoint by just rewriting the grid-template-areas string — no need to touch the HTML at all.

@media (max-width: 768px) {
  body {
    grid-template-areas:
      "header"
      "main"
      "sidebar"
      "footer";
  }
}
Pro Tip

Open Chrome DevTools → Elements → click the grid badge next to your container — it shows a visual overlay of your named areas directly on the page.

Trick

Align your area names vertically in the CSS string — it makes the blueprint readable at a glance:

/* Hard to read */
grid-template-areas: "header header" "nav nav" "sidebar main" "footer footer";

/* Easy to read */
grid-template-areas:
  "header  header"
  "nav     nav"
  "sidebar main"
  "footer  footer";
02

grid-template-columns

Pro Tip

Use auto-fill with minmax() to create a responsive grid with no media queries — it automatically adds or removes columns based on available space.

.cards {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
  /* Cards are at least 250px wide — as many as fit per row */
}
Pro Tip

auto-fill vs auto-fit — auto-fill keeps empty columns, auto-fit collapses them and stretches existing items to fill the space. For card grids, use auto-fill.

Pro Tip

The holy grail container pattern — full width on mobile, capped on desktop, always centered:

.container {
  width: min(90%, 1200px);
  margin-inline: auto; /* centers horizontally */
}
Trick

Make one column take up all remaining space using 1fr while others are fixed — perfect for a sidebar + main content layout:

grid-template-columns: 260px 1fr 300px;
/* sidebar | grows to fill space | widgets */
Trick

Span an item across multiple columns using grid-column: span N — no need to define the exact start and end:

.featured-card {
  grid-column: span 2; /* takes up 2 columns */
}
03

grid-template-rows

Pro Tip

Use 1fr on the main content row to make it fill all remaining vertical space — perfect for a sticky footer layout:

body {
  display: grid;
  grid-template-rows: 70px 1fr 60px;
  min-height: 100vh;
  /* header | content fills space | footer sticks to bottom */
}
Pro Tip

Never use height: 100vh on mobile — the browser's address bar shrinks and expands, causing layout jumps. Use min-height: 100svh instead (svh = small viewport height).

.hero {
  min-height: 100svh; /* safe on mobile browsers */
}
Trick

Span an item across multiple rows using grid-row: span N:

.tall-card {
  grid-row: span 2; /* takes up 2 rows height */
}
Trick

Use subgrid to align nested grid children with the outer grid — useful for complex card layouts where inner content needs to align across cards.

04

grid-auto-rows

Pro Tip

The safest and most used value is minmax(0, auto) — rows never collapse to zero but grow freely with content:

grid-auto-rows: minmax(0, auto);
Pro Tip

For masonry-style galleries, combine grid-auto-rows with small row values and grid-row: span N on items to create Pinterest-like layouts.

.masonry {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-auto-rows: 10px; /* tiny row unit */
}

.tall-item   { grid-row: span 30; }
.medium-item { grid-row: span 20; }
.short-item  { grid-row: span 15; }
Trick

Combine grid-auto-flow: dense with auto rows to automatically fill empty gaps in your grid — items rearrange to fill holes:

.grid {
  grid-auto-flow: dense; /* fills gaps automatically */
}
05

Flexbox

Pro Tip

The fastest way to perfectly center anything — horizontally and vertically — in 3 lines:

.center-me {
  display: flex;
  align-items: center;
  justify-content: center;
}
Pro Tip

Use margin-left: auto on a flex child to push it all the way to the right — perfect for a navbar with a button that needs to be on the far right:

nav {
  display: flex;
  align-items: center;
}

.cta-button {
  margin-left: auto; /* pushes to the far right */
}
Pro Tip

Use flex: 1 on a child to make it fill all remaining space. Two children with flex: 1 = equal width. One with flex: 2 and one with flex: 1 = 2/3 and 1/3 split.

Trick

Sticky footer with flexbox — push footer to the bottom even when content is short:

body {
  display: flex;
  flex-direction: column;
  min-height: 100vh;
}

main {
  flex: 1; /* grows to push footer down */
}
Trick

Use flex-wrap: wrap + flex: 1 1 250px for a simple responsive grid without media queries — items wrap to the next line when they can't fit:

.cards {
  display: flex;
  flex-wrap: wrap;
  gap: 16px;
}

.card {
  flex: 1 1 250px; /* grow, shrink, base 250px */
}
06

Gap

Pro Tip

Define your gap as a CSS variable so you can change spacing across the whole layout from one place:

:root { --gap: 24px; }

.grid { gap: var(--gap); }
.flex { gap: var(--gap); }
Pro Tip

Use different row-gap and column-gap for breathing room — more vertical space between rows often looks better than equal gaps:

.cards {
  row-gap: 32px;    /* more space between rows */
  column-gap: 16px; /* tighter between columns */
}
Trick

gap works on flex containers too — much cleaner than adding margin to every child item.

07

Media Queries

Pro Tip

Store breakpoints in CSS variables at the top of your file so they're consistent and easy to update:

/* At the top of your CSS */
/* --mobile: 768px  */
/* --tablet: 1024px */

@media (max-width: 768px)  { /* mobile */ }
@media (max-width: 1024px) { /* tablet */ }
Pro Tip

Use prefers-color-scheme to automatically apply dark mode based on the user's OS setting — no JavaScript toggle needed:

@media (prefers-color-scheme: dark) {
  body {
    background: #1a1a2e;
    color: #e0e0f0;
  }
}
Pro Tip

Use prefers-reduced-motion to disable animations for users who have motion sensitivity enabled in their OS:

@media (prefers-reduced-motion: reduce) {
  * {
    animation: none !important;
    transition: none !important;
  }
}
Trick

Test all your breakpoints at once using Chrome DevTools — open Device Toolbar, drag the edge of the viewport slowly, and watch your layout respond in real time.

08

Position

Pro Tip

Use inset: 0 instead of writing top: 0; right: 0; bottom: 0; left: 0 — it's a shorthand that covers all 4 sides at once. Perfect for overlays:

.overlay {
  position: fixed;
  inset: 0; /* covers the entire screen */
  background: rgba(0,0,0,0.5);
}
Pro Tip

Center an absolute element inside its parent using translate — works regardless of the element's size:

.parent { position: relative; }

.child {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
  /* perfectly centered every time */
}
Trick

Use position: sticky on table headers to keep them visible while scrolling through long data tables — works without any JavaScript:

th {
  position: sticky;
  top: 0;
  background: white; /* needed to cover rows behind it */
  z-index: 1;
}
Trick

Use position: sticky with align-self: start on a sidebar inside a grid to make it stick while the main content scrolls past:

.sidebar {
  position: sticky;
  top: 80px; /* offset for fixed navbar */
  align-self: start;
}
09

Z-index

Pro Tip

Define all your z-index values as CSS variables at the top of your file — this prevents the "z-index wars" where everyone keeps adding higher and higher numbers:

:root {
  --z-below:    -1;
  --z-base:      1;
  --z-dropdown: 100;
  --z-sticky:   200;
  --z-modal:    300;
  --z-toast:    400;
  --z-tooltip:  500;
}
Pro Tip

Use isolation: isolate on a container to create a new stacking context without setting a specific z-index — useful when a component's children are interfering with outside elements:

.card {
  isolation: isolate; /* children stack only within this card */
}
Trick

Use z-index: -1 to place decorative elements behind their parent — perfect for background effects and pseudo-elements:

.hero::before {
  content: '';
  position: absolute;
  inset: 0;
  background: url('pattern.png');
  z-index: -1; /* behind hero content */
}
10

Transitions & Animations

Pro Tip

Only animate transform and opacity — they are GPU-accelerated and never cause layout reflow. Everything else is slow:

/* FAST — GPU accelerated */
.box { transition: transform 0.3s ease, opacity 0.3s ease; }

/* SLOW — causes layout reflow */
.box { transition: width 0.3s, height 0.3s, top 0.3s; }
Pro Tip

Use animation-delay with calc() to stagger animations across multiple items — no JavaScript needed:

.card:nth-child(1) { animation-delay: 0ms; }
.card:nth-child(2) { animation-delay: 100ms; }
.card:nth-child(3) { animation-delay: 200ms; }

/* Or with CSS variables */
.card { animation-delay: calc(var(--i) * 100ms); }
Pro Tip

Use will-change: transform to hint the browser to prepare GPU acceleration before an animation starts — reduces jank on complex animations:

.animated-card {
  will-change: transform;
}
Trick

Create a smooth loading skeleton effect with just CSS animation:

@keyframes shimmer {
  0%   { background-position: -200% 0; }
  100% { background-position: 200% 0; }
}

.skeleton {
  background: linear-gradient(90deg,
    #2a2a45 25%, #3a3a55 50%, #2a2a45 75%);
  background-size: 200% 100%;
  animation: shimmer 1.5s infinite;
}
11

CSS Variables

Pro Tip

Provide a fallback value in var() in case the variable isn't defined — prevents invisible broken styles:

color: var(--primary, #e94560);
/* uses --primary, falls back to #e94560 if not set */
Pro Tip

CSS variables can be updated with JavaScript — making them perfect for dynamic theming, user-controlled settings, or animation values:

// Change a CSS variable with JavaScript
document.documentElement.style
  .setProperty('--primary', '#00b894');

// Read a CSS variable with JavaScript
const color = getComputedStyle(document.documentElement)
  .getPropertyValue('--primary');
Trick

Use CSS variables for responsive spacing — define one set and change only the values at breakpoints, not every property:

:root {
  --space-lg: 80px;
  --space-md: 40px;
}

@media (max-width: 768px) {
  :root {
    --space-lg: 40px;
    --space-md: 20px;
  }
}

/* Everything using these variables updates automatically */
section { padding: var(--space-lg) var(--space-md); }
12

Overflow

Pro Tip

Use overflow: clip instead of hidden when you don't want to create a new scroll context — hidden makes the element scrollable via JavaScript, clip truly clips with no side effects:

.card {
  overflow: clip; /* true clip, no scroll context */
}
Pro Tip

Create a horizontal scrollable row of cards (like Netflix categories) using overflow:

.row {
  display: flex;
  gap: 16px;
  overflow-x: auto;
  scroll-snap-type: x mandatory; /* snaps to each card */
  padding-bottom: 8px; /* space for scrollbar */
}

.card {
  flex-shrink: 0; /* cards never shrink */
  scroll-snap-align: start;
}
Trick

Hide the scrollbar visually while keeping scroll functionality — common in modern UIs:

.scroll-container {
  overflow-x: auto;
  scrollbar-width: none; /* Firefox */
}

.scroll-container::-webkit-scrollbar {
  display: none; /* Chrome, Safari */
}
13

JavaScript DOM

Pro Tip

Use event delegation — attach one listener to the parent instead of one on every child. Faster and works for dynamically added elements:

/* Bad — one listener per card */
cards.forEach(card => card.addEventListener('click', fn));

/* Good — one listener on parent */
document.querySelector('.cards')
  .addEventListener('click', (e) => {
    if (e.target.closest('.card')) {
      // handle card click
    }
  });
Pro Tip

Use dataset to store extra data on HTML elements — access it cleanly in JavaScript without needing extra variables:

<button data-id="42" data-action="delete">Delete</button>

btn.addEventListener('click', (e) => {
  console.log(e.target.dataset.id);     // "42"
  console.log(e.target.dataset.action); // "delete"
});
Trick

Debounce scroll and resize events — they fire hundreds of times per second. Debouncing waits until the user stops before running your function:

function debounce(fn, delay) {
  let timer;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn(...args), delay);
  };
}

window.addEventListener('resize', debounce(() => {
  // runs only after user stops resizing
}, 200));
14

clamp / min / max

Pro Tip

Build a complete fluid typography scale using only clamp() — no media queries for fonts ever again:

:root {
  --text-sm:   clamp(0.8rem,  1.5vw, 1rem);
  --text-base: clamp(1rem,    2vw,   1.2rem);
  --text-lg:   clamp(1.2rem,  2.5vw, 1.5rem);
  --text-xl:   clamp(1.5rem,  3.5vw, 2.5rem);
  --text-2xl:  clamp(2rem,    5vw,   4rem);
}
Pro Tip

Use clamp() for responsive padding that breathes on large screens but stays tight on mobile:

.section {
  padding-block:  clamp(40px, 10vh, 140px);
  padding-inline: clamp(16px, 8vw, 120px);
}
Trick

The perfect container — full width on mobile, never too wide on desktop, always centered:

.container {
  width: min(90%, 1200px);
  margin-inline: auto;
  padding-inline: clamp(16px, 5vw, 40px);
}
15

HTML Semantics

Pro Tip

Use <button> for actions and <a> for navigation — never the other way around. Buttons submit, toggle, and trigger actions. Links go somewhere.

Pro Tip

Use <picture> instead of <img> when you need different images for different screen sizes — loads the right size, saves bandwidth:

<picture>
  <source media="(max-width: 768px)" srcset="hero-mobile.webp">
  <source media="(min-width: 769px)" srcset="hero-desktop.webp">
  <img src="hero-desktop.webp" alt="Hero" />
</picture>
Trick

Use <details> and <summary> for a native accordion/FAQ — no JavaScript needed:

<details>
  <summary>What is your return policy?</summary>
  <p>You can return any item within 30 days...</p>
</details>
16

Forms & Inputs

Pro Tip

Use autocomplete attributes — browsers auto-fill common fields, making forms faster for users:

<input type="email" autocomplete="email" />
<input type="text"  autocomplete="name" />
<input type="tel"   autocomplete="tel" />
Pro Tip

Use inputmode on mobile to show the right keyboard for the input type:

<input inputmode="numeric"   /> <!-- number pad -->
<input inputmode="email"     /> <!-- @ key visible -->
<input inputmode="tel"       /> <!-- phone pad -->
<input inputmode="url"       /> <!-- .com key visible -->
Trick

Style a custom checkbox without any JavaScript — using only CSS:

<input type="checkbox" id="agree" />
<label for="agree">I agree</label>

input[type="checkbox"] { display: none; }

input:checked + label::before {
  background: #e94560;
}
17

CSS Pseudo-classes

Pro Tip

Use ::before and ::after pseudo-elements to add decorative content without touching HTML — great for badges, quotation marks, decorative lines:

.section-title::after {
  content: '';
  display: block;
  width: 60px;
  height: 3px;
  background: #e94560;
  margin-top: 8px;
}
Pro Tip

Use :is() to group selectors cleanly — reduces repetition:

/* Old way */
h1, h2, h3, h4 { font-weight: 700; }

/* New way */
:is(h1, h2, h3, h4) { font-weight: 700; }
Trick

Use :has() — the parent selector — to style a parent based on what's inside it. Very powerful for conditional styling:

/* Style a card differently when it contains an image */
.card:has(img) {
  padding: 0;
}

/* Style a form when it has an invalid input */
form:has(input:invalid) .submit-btn {
  opacity: 0.5;
}

Accessibility

Pro Tip

Add a "Skip to main content" link as the very first element — keyboard and screen reader users can jump past the navbar straight to the content:

<a href="#main" class="skip-link">Skip to main content</a>

.skip-link {
  position: absolute;
  top: -100px;
  left: 0;
}

.skip-link:focus {
  top: 0; /* appears only when focused via keyboard */
}
Pro Tip

Use aria-live="polite" on areas that update dynamically — screen readers announce the change automatically:

<div aria-live="polite" id="status"></div>

// When something changes:
document.getElementById('status').textContent =
  'Form submitted successfully!';
Trick

Use the free axe DevTools browser extension to instantly audit any page for accessibility issues — it highlights every problem with an explanation of how to fix it.

SEO Basics

Pro Tip

Add Open Graph meta tags so your page looks great when shared on social media — Facebook, Twitter, WhatsApp all read these:

<meta property="og:title"       content="BrewBolt Coffee" />
<meta property="og:description" content="Fresh coffee delivered." />
<meta property="og:image"       content="https://..." />
<meta property="og:url"         content="https://brewbolt.com" />
Pro Tip

Add a favicon — it shows in the browser tab and makes your site look professional and finished:

<link rel="icon" type="image/png" href="favicon.png" />
Trick

Use Google's free Rich Results Test tool to check if your page qualifies for rich snippets in search results — stars, prices, FAQs showing directly in Google.

20

Performance

Pro Tip

Preload your most important above-the-fold resources — hero image, main font — so the browser fetches them immediately:

<link rel="preload" as="image" href="hero.webp" />
<link rel="preload" as="font"
      href="Inter.woff2" crossorigin />
Pro Tip

Add width and height attributes to every image — prevents Cumulative Layout Shift (CLS) where the page jumps as images load:

<img src="product.jpg" alt="Product"
     width="400" height="300"
     loading="lazy" />
Trick

Use content-visibility: auto on off-screen sections — the browser skips rendering them until the user scrolls near. Can make very long pages 5x faster:

.below-fold-section {
  content-visibility: auto;
  contain-intrinsic-size: 0 500px; /* estimated height */
}
21

Fonts & Icons

Pro Tip

Use font-display: swap to show text immediately in a fallback font while the custom font loads — prevents invisible text during load:

/* Google Fonts adds this automatically when you use ?display=swap */
https://fonts.googleapis.com/css2?family=Inter&display=swap
Pro Tip

Use CSS font-size and color to style Font Awesome icons — they are just text characters that look like icons:

.icon {
  font-size: 2rem;
  color: var(--primary);
  transition: color 0.2s;
}

.icon:hover { color: var(--dark); }
Trick

Use SVG icons instead of Font Awesome for better performance and control — inline SVG can be styled with CSS directly and loads with zero extra requests.

JavaScript Beyond DOM

Pro Tip

Use async/await instead of .then() chains — much easier to read and debug:

/* Old way */
fetch(url).then(r => r.json()).then(data => console.log(data));

/* Better way */
async function getData() {
  const response = await fetch(url);
  const data = await response.json();
  console.log(data);
}
Pro Tip

Always wrap fetch calls in try/catch — network requests fail silently without it:

async function getData() {
  try {
    const res = await fetch(url);
    const data = await res.json();
    return data;
  } catch (error) {
    console.error('Fetch failed:', error);
  }
}
Trick

Use sessionStorage for data that should clear when the tab closes, and localStorage for data that persists between visits — like a user's dark mode preference.

VS Code Workflow

Pro Tip

Use Ctrl + P to instantly open any file by name — much faster than clicking through the file explorer:

Ctrl + P           → open file by name
Ctrl + Shift + F   → search text across ALL files
Ctrl + G           → go to specific line number
Pro Tip

Use multi-cursor editing to change the same thing in multiple places at once:

Ctrl + D           → select next match of current word
Ctrl + Shift + L   → select ALL matches at once
Alt + Click        → manually place extra cursors anywhere
Trick

Split your editor into two panels — HTML on the left, CSS on the right — so you never need to switch tabs:

Ctrl + \           → split editor
Ctrl + 1 / 2       → focus left or right panel
Trick

Fold and unfold code blocks to hide sections you're not working on — keeps your editor clean:

Ctrl + Shift + [   → fold current block
Ctrl + Shift + ]   → unfold current block
Ctrl + K, Ctrl + 0 → fold all
Ctrl + K, Ctrl + J → unfold all

Browser DevTools

Pro Tip

Use the computed tab in DevTools to see the final calculated value of any CSS property — useful when you don't know which rule is winning:

Elements → Computed tab → search for any property
→ see the exact value the browser is using
→ click the arrow to jump to the source CSS rule
Pro Tip

Use $0 in the Console to reference the currently selected element in the Elements panel — run any JavaScript on it instantly:

// Select an element in Elements panel, then in Console:
$0.style.background = 'red';
$0.classList.add('highlight');
console.log($0.offsetWidth); // get its width
Trick

Use the Coverage tool to find unused CSS and JavaScript — open DevTools → More tools → Coverage → reload. Red = unused code you can delete.

Trick

Throttle network speed to test how your site loads on a slow connection — Network tab → No throttling → Slow 3G. Shows how real users on mobile networks experience your site.

Git & Version Control

Pro Tip

Always create a .gitignore file at the start of every project — prevents accidentally uploading large or sensitive files:

# .gitignore file
node_modules/
.env
.DS_Store
*.log
dist/
Pro Tip

Write commit messages in the present tense, describing what the commit does — not what you did:

# Bad
git commit -m "fixed the navbar"
git commit -m "I changed the colors"

# Good
git commit -m "Fix navbar collapse on mobile"
git commit -m "Update brand colors to new palette"
Pro Tip

Use git stash to temporarily save uncommitted changes when you need to switch branches quickly:

git stash          # saves current changes
git checkout main  # switch branch safely
git stash pop      # restore your saved changes
Trick

Use git log --oneline --graph to see a visual tree of all your branches and commits — great for understanding your project history:

git log --oneline --graph --all

Hosting & Deployment

Pro Tip

Always name your main page index.html — web servers automatically serve this file when someone visits your URL with no filename:

yoursite.com/       → serves index.html
yoursite.com/about → serves about/index.html
Pro Tip

Add a _redirects file to Netlify to handle 404 pages and URL redirects:

# _redirects file in your project root
/old-page    /new-page    301
/*           /index.html  200
Pro Tip

Use Netlify's free form handling — add netlify attribute to your HTML form and it handles submissions without any backend code:

<form name="contact" method="POST" data-netlify="true">
  <input type="text" name="name" />
  <button type="submit">Send</button>
</form>
Trick

Set up a custom 404 page — create a file named 404.html in your root. Netlify and GitHub Pages automatically serve it when a page isn't found — much better than a blank browser error.

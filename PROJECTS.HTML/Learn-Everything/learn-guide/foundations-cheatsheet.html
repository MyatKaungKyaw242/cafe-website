The Box Model

The 4 Layers

Every element is a box. From inside out: content → padding → border → margin.

.box {
  /* Content: set by width/height */
  width: 200px;
  height: 100px;

  /* Padding: space INSIDE the border */
  padding: 16px;
  padding: 10px 20px;     /* top/bottom  left/right */
  padding: 10px 20px 10px 20px; /* top right bottom left */

  /* Border: the visible edge */
  border: 2px solid #fdcb6e;

  /* Margin: space OUTSIDE the border */
  margin: 24px;
  margin: 0 auto; /* center horizontally */
}

box-sizing: border-box

Makes width include padding and border. Put this on every project — always.

/* ✅ First rule in every CSS file */
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

/* Without it — width EXPANDS with padding */
.card { width:300px; padding:20px; }
/* real width = 300 + 40 = 340px ❌ */

/* With border-box — width STAYS fixed */
.card { width:300px; padding:20px; }
/* real width = 300px ✓ */

Margin Shorthand

1 value = all sides. 2 values = top/bottom, left/right. 4 values = clockwise.

margin: 20px;              /* all 4 sides */
margin: 20px 40px;         /* top&bottom  left&right */
margin: 10px 20px 30px 40px; /* top right bottom left */

/* Center a block element */
margin: 0 auto;

/* Individual sides */
margin-top:    10px;
margin-right:  20px;
margin-bottom: 10px;
margin-left:   20px;

/* Same shorthand works for padding */
padding: 16px 24px;

Margin Collapse

Vertical margins between siblings collapse — only the larger one applies.

/* Two siblings — margins collapse */
.first  { margin-bottom: 20px; }
.second { margin-top: 30px; }
/* Space between them = 30px, NOT 50px */

/* Fix: use padding instead of margin
   on the parent, or use gap in flex/grid */
.parent { padding-bottom: 20px; }

/* Horizontal margins NEVER collapse */
.left  { margin-right: 20px; }
.right { margin-left: 10px; }
/* Space between them = 30px ✓ */
CSS Units

Absolute Units

Fixed size — doesn't scale with anything. Use px for small fixed values.

/* px — pixels, most common absolute unit */
border: 1px solid #333;
border-radius: 4px;
box-shadow: 0 2px 8px rgba(0,0,0,0.2);

/* When to use px: */
/* ✓ Borders (1px, 2px) */
/* ✓ Border-radius */
/* ✓ Box shadows */
/* ✓ Tiny precise values */

/* ✗ Font sizes — use rem instead */
/* ✗ Large spacing — use rem instead */

Relative Units

Scale with something else — the root, the parent, or the viewport.

/* rem — relative to root (html) font size */
/* 1rem = 16px by default */
font-size: 1rem;    /* 16px */
font-size: 1.5rem;  /* 24px */
font-size: 0.875rem;/* 14px */

/* em — relative to PARENT font size */
/* Compounds! Avoid for font sizes */
font-size: 1.2em; /* 1.2 × parent size */

/* % — relative to parent's value */
width: 100%;       /* full parent width */
width: 50%;        /* half parent width */

/* vh / vw — % of viewport */
height: 100vh;     /* full screen height */
width: 100vw;      /* full screen width */

clamp() — Responsive Sizing

Set a min, preferred, and max value in one line. Perfect for fluid typography.

/* clamp(min, preferred, max) */
font-size: clamp(1rem, 2.5vw, 2rem);
/* min: never smaller than 1rem (16px) */
/* preferred: 2.5% of viewport width */
/* max: never larger than 2rem (32px) */

/* Common fluid typography scale */
h1 { font-size: clamp(1.8rem, 5vw, 3.5rem); }
h2 { font-size: clamp(1.4rem, 3.5vw, 2.5rem); }
h3 { font-size: clamp(1.1rem, 2.5vw, 1.8rem); }
p  { font-size: clamp(1rem,   1.5vw, 1.125rem); }

/* Also works for spacing */
padding: clamp(1rem, 5%, 3rem);

When to Use Which

Quick reference for choosing the right unit every time.

  • Font sizes → rem (scales with user settings)
  • Fluid font sizes → clamp()
  • Spacing (padding/margin) → rem
  • Full-width containers → % or 100vw
  • Full-height sections → 100vh
  • Borders → px
  • Border-radius → px or % (50% = circle)
  • Max-width on content → px e.g. max-width: 1100px
Specificity & The Cascade

Specificity Score

Each selector type has a weight. Higher score wins the conflict.

/* Score: (inline, ID, class, tag) */

p               { }  /* 0,0,0,1 — tag */
.card           { }  /* 0,0,1,0 — class */
.card p         { }  /* 0,0,1,1 — class + tag */
.card .title    { }  /* 0,0,2,0 — two classes */
#header         { }  /* 0,1,0,0 — ID */
style="..."        /* 1,0,0,0 — inline */
color: red !important; /* wins everything (avoid!) */

/* Higher score always wins, regardless of order */
#btn            { color: red; }   /* score 0,1,0,0 */
.btn.primary    { color: blue; }  /* score 0,0,2,0 */
/* Result: red — ID beats two classes */

Cascade — Last Rule Wins

When specificity is equal, the rule that appears last in the file wins.

/* Both have the same specificity (0,0,1,0) */
.btn { background: blue; }   /* written first */
.btn { background: green; }  /* written after */
/* Result: green — last wins ✓ */

/* This is why order matters in your CSS */
/* Put base/default styles first */
/* Put specific overrides after */

/* Example: */
.card { padding: 16px; }         /* base */
.card.featured { padding: 24px; } /* override — comes after */

Selector Types

Quick reference for common selectors and what they target.

/* Tag — targets all elements of that type */
p { }  h1 { }  button { }

/* Class — targets elements with that class */
.card { }  .btn-primary { }

/* ID — targets one specific element (avoid for CSS) */
#header { }

/* Descendant — targets .title inside .card */
.card .title { }

/* Child — direct children only */
.nav > li { }

/* Pseudo-class — specific state */
a:hover { }
input:focus { }
li:first-child { }
li:nth-child(2) { }

/* Pseudo-element — part of an element */
p::first-line { }
li::before { content: '›'; }

The Golden Rules

Follow these and specificity problems disappear.

  • Style with classes — not IDs, not tags alone
  • Never use !important — fix the specificity instead
  • Keep selectors short — .card-title beats .card div h2 span
  • Order matters — base styles first, overrides last
  • Use DevTools — crossed-out styles in the Styles panel = being overridden
  • BEM naming helps: .card, .card__title, .card--featured
Color Formats

Color Formats

CSS supports multiple formats. Use hex for solid colors, rgba for transparency.

/* Named colors — avoid, limited options */
color: red;
color: cornflowerblue;

/* Hex — most common for solid colors */
color: #e94560;      /* 6 digits */
color: #fff;         /* short = #ffffff */
color: #e9456080;    /* 8 digits = includes opacity */

/* RGB — useful when you need to vary opacity */
color: rgb(233, 69, 96);
color: rgba(233, 69, 96, 0.5); /* 50% transparent */

/* HSL — most intuitive for adjusting colors */
color: hsl(350, 80%, 59%);
color: hsla(350, 80%, 59%, 0.5);

/* Modern — space-separated, no commas */
color: rgb(233 69 96 / 50%);

CSS Variables for Colors

Define your palette once in :root. Change one line to update the whole site.

:root {
  /* Brand palette */
  --primary:   #e94560;
  --secondary: #48cae4;
  --accent:    #fdcb6e;

  /* Backgrounds */
  --bg:        #0f0f1a;
  --surface:   #1a1a2e;

  /* Borders */
  --border:    #2a2a45;

  /* Text */
  --text:      #e0e0f0;
  --muted:     #888aaa;
}

/* Use everywhere */
.card     { background: var(--surface); }
.btn      { background: var(--primary); }
.subtitle { color: var(--muted); }

Opacity & Transparency

Multiple ways to make elements or colors transparent.

/* rgba — transparent COLOR only */
background: rgba(233, 69, 96, 0.1);
/* element content stays fully visible */

/* opacity — transparent ENTIRE ELEMENT */
.faded { opacity: 0.5; }
/* everything including children fades */

/* Hex with alpha */
background: #e9456020; /* last 2 digits = opacity */
/* 00=0%  40=25%  80=50%  bf=75%  ff=100% */

/* CSS variable trick — define color as RGB channels */
:root { --primary-rgb: 233, 69, 96; }
background: rgba(var(--primary-rgb), 0.1);
background: rgba(var(--primary-rgb), 0.5);

Common Color Pairs

Tried-and-tested combinations that always look good together.

  • Dark theme: bg #0f0f1a, text #e0e0f0
  • Light theme: bg #f8f9fa, text #1a1a2e
  • Danger/Error: #e94560 on dark, #d63031 on light
  • Success: #55efc4 on dark, #00b894 on light
  • Warning: #fdcb6e on dark, #e17055 on light
  • Info: #48cae4 on dark, #0984e3 on light
  • Subtle borders: white at 8–15% opacity on dark themes
Typography

Font Stack

Always provide fallback fonts. The browser uses the first available one.

/* System font stack — fast, no download needed */
font-family: 'Segoe UI', Arial, sans-serif;

/* Common stacks */
font-family: Georgia, 'Times New Roman', serif;  /* classic serif */
font-family: 'Courier New', Courier, monospace;  /* code */

/* Google Fonts — link in <head> first */
@import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;600;800&display=swap');
font-family: 'Inter', sans-serif;

/* Variable font — one file, many weights */
font-weight: 100 900; /* any weight in range */

Type Scale

A consistent scale makes headings feel intentional, not random.

/* Base: 1rem = 16px */
/* Scale ratio: ~1.25 (Major Third) */

.text-xs   { font-size: 0.75rem;  } /* 12px */
.text-sm   { font-size: 0.875rem; } /* 14px */
.text-base { font-size: 1rem;     } /* 16px */
.text-lg   { font-size: 1.125rem; } /* 18px */
.text-xl   { font-size: 1.25rem;  } /* 20px */
.text-2xl  { font-size: 1.5rem;   } /* 24px */
.text-3xl  { font-size: 1.875rem; } /* 30px */
.text-4xl  { font-size: 2.25rem;  } /* 36px */

/* Or use clamp() for fluid scaling */
h1 { font-size: clamp(1.8rem, 5vw, 3rem); }

Readability Properties

The small tweaks that separate professional type from beginner type.

/* ✅ Set these globally on body */
body {
  line-height: 1.6;        /* comfortable for body text */
  font-size: 1rem;
  color: var(--text);
}

/* Headings — tighter, heavier */
h1, h2, h3 {
  line-height: 1.2;
  font-weight: 800;
  letter-spacing: -0.5px; /* slightly tighter at large sizes */
}

/* Long-form text — optimal reading width */
.prose {
  max-width: 65ch; /* ~65 characters per line */
}

/* Labels, badges — spaced out */
.label {
  font-size: 0.75rem;
  font-weight: 700;
  letter-spacing: 1.5px;
  text-transform: uppercase;
}

Text Utilities

Common text styling properties you'll use constantly.

/* Alignment */
text-align: left | center | right | justify;

/* Decoration */
text-decoration: none;          /* remove underline */
text-decoration: underline;

/* Transform */
text-transform: uppercase;
text-transform: capitalize;

/* Overflow — truncate long text */
.truncate {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis; /* adds "..." */
}

/* Weight */
font-weight: 400;  /* normal */
font-weight: 600;  /* semi-bold */
font-weight: 700;  /* bold */
font-weight: 900;  /* black */
Planning & Debugging Quick Reference

Layout Planning Questions

Answer these 3 questions before writing any CSS.

  • What are the sections? Header, sidebar, main, footer
  • How many columns? 1, 2, 3? Fixed sidebar or flexible?
  • What happens on mobile? Stack vertically? Hide sidebar?
  • Draw boxes on paper or in a notes app first
  • Start with the outermost container, work inward
  • Use Grid for the page layout, Flexbox for components
  • Add outline: 1px solid red on elements to see their real size

Project File Structure

Standard structure for a clean, self-contained project.

project-name/
├── index.html         ← entry point
├── css/
│   └── style.css      ← all styles
├── js/
│   └── main.js        ← all scripts
└── images/
    └── logo.png

<!-- In HTML — relative paths always -->
<link rel="stylesheet" href="css/style.css" />
<script src="js/main.js"></script>
<img src="images/logo.png" alt="Logo" />

<!-- Script: bottom of body OR defer in head -->
<script src="js/main.js" defer></script>

CSS Not Working? Checklist

Work through this in order — one of these is almost always the cause.

  • Is the CSS file linked? Check the <link> href path
  • Typo in class name? Compare HTML and CSS exactly
  • Being overridden? Open DevTools → Styles panel → look for crossed-out rules
  • Wrong element selected? Add a temp color to confirm the selector works
  • Specificity battle? Simplify the selector or use a more specific class
  • Missing box-sizing: border-box? Add the * reset
  • Flexbox on wrong element? Parent needs display: flex, not the child

DevTools — Must-Know Shortcuts

The most useful DevTools actions for a beginner.

  • F12 — open/close DevTools
  • Right-click → Inspect — jump to that element
  • Ctrl+Shift+C — click to inspect any element
  • Styles panel — see all CSS on selected element
  • Crossed-out style — being overridden, click to see by what
  • Click any value — edit it live (resets on refresh)
  • Console tab — errors + console.log output
  • Network tab — check if files loaded (red = 404)

The Debugging Process

A repeatable system for fixing any bug, any language.

/* Step 1 — Read the error message */
/* Error messages tell you: what went wrong + file + line number */
/* Don't skip this. Read every word carefully. */

/* Step 2 — Reproduce it reliably */
/* Can you make it happen on demand? If yes, you can fix it. */

/* Step 3 — Isolate it */
/* Comment out half the code. Still broken? The bug is in the remaining half. */
/* Repeat until you've narrowed it to one function or one rule. */

/* Step 4 — Check your assumptions */
console.log("value is:", myVariable); // is it what you think it is?

/* Step 5 — Google the exact error message in quotes */
// "Cannot read properties of null (reading 'innerHTML')"
// Someone has had this exact error before.

/* Step 6 — Take a break if stuck for 30+ minutes */
/* Fresh eyes catch what tired eyes miss. Genuinely. */

Claude Cheatsheet

Your quick reference guide to get the best results from Claude

🔨

BUILD Commands

Basic

"Build a [type] website with [style] using [tech]" "Create a [dark/light/colorful] [navbar/hero/card/footer]" "Make a full page layout for [purpose]" "Build only the [section] part"

Advanced

"Build a responsive [website] that works on mobile and desktop" "Create a [website] with animations and hover effects" "Make a [dark/glassmorphism/neumorphism] themed [component]" "Build a [website] inspired by [brand/style] like Airbnb / Apple" "Create a multi-section landing page with [hero, about, contact]"
💡 Tip: The more detail you give, the better the result. Mention colors, fonts, layout style, and purpose.
✏️

EDIT Commands

Basic

"Change [this] to [that] in my [filename]" "Add [feature] to my [filename]" "Fix the [problem] in my code" "Update the colors / fonts / spacing"

Advanced

"Refactor my CSS to use variables" "Make my [filename] responsive for mobile" "Clean up my code and remove unused styles" "Add smooth scroll and transition animations" "Convert my inline styles to a separate CSS file" "Rename all my classes to follow BEM naming convention"
💡 Tip: Always mention the filename so Claude edits the right file.
📚

LEARN Commands

Basic

"Explain [topic] like I'm a beginner" "Explain [topic] with a code example" "What is the difference between [A] and [B]?" "When should I use [A] vs [B]?" "Show me how [topic] works step by step"

Advanced

"Give me a deep explanation of [topic] with real world examples" "What are the best practices for [topic]?" "What are common mistakes beginners make with [topic]?" "Explain [topic] visually using ASCII or a table" "Summarize [topic] in 5 bullet points" "Give me a mini project to practice [topic]" "Quiz me on [topic]"
💡 Tip: Say "explain like I'm a beginner" or "give a deep explanation" to control how detailed the answer is.
🔁

FOLLOW-UP Commands

Refinement

"Redo it" "Make it simpler" "Make it more complex / advanced" "Give me another version" "Make it look more professional" "Make it more minimal / clean"

Understanding

"Why does this work?" "Break down this code line by line" "What does [specific line] do?" "Is there a better way to write this?" "What would happen if I removed [X]?"

Adding on

"Add [X] to it" "Now add a mobile version" "Add JavaScript to make [X] interactive" "Add hover effects and animations"
📁

FILE Commands

Managing Files

"Put this code into my [filename]" "Make separate HTML and CSS files" "Save this into my [folder name]" "Show me what's in my [folder]" "Create a new file called [name] in [folder]" "Link my CSS file to my HTML file"

Reading Files

"Read my [filename] and explain what it does" "Check my [filename] for any mistakes or bugs" "Compare my [file A] and [file B]"
💡 Tip: Claude can only access files inside the folder you opened it from in VS Code.
🐛

DEBUG Commands

Finding Problems

"My [X] is not working, can you fix it?" "Why is my layout broken?" "My CSS is not applying, what's wrong?" "Find the bug in my [filename]"

Specific Issues

"My flexbox / grid is not centering correctly" "My navbar is overlapping the content, fix it" "My website is not responsive on mobile" "My font / color is not showing correctly" "My button click is not working"
💡 Tip: Paste your broken code directly into the chat for the fastest fix.
🗂️

GIT Commands

Basic Git Help

"What is Git and why do we use it?" "How do I commit and push my files to GitHub?" "Show me my recent git commits" "What files have I changed?"

Actions

"Commit my changes with message [message]" "Create a pull request for my changes" "Show me what branch I'm on"
🎯

PRACTICE Commands

Getting Exercises

"Give me a beginner exercise for [topic]" "Give me a challenge to practice [CSS Grid / Flexbox / JS]" "Create a mini project I can build to learn [topic]" "Quiz me on [topic] with 5 questions"

Reviewing Your Work

"Review my code and give me feedback" "Grade my [filename] out of 10 and tell me what to improve" "What am I missing in my [filename]?" "How would a professional improve this?"

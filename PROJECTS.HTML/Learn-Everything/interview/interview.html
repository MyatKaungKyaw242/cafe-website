HTML Questions

Semantics, accessibility, structure

6 Questions
What is the difference between <div> and semantic tags like <article>, <section>, <nav>?

<div> is a generic container with no meaning. Semantic tags tell the browser — and developers — what the content IS, not just how it's grouped.

  • <header> — introductory content for a page or section
  • <nav> — navigation links
  • <main> — the main content (only one per page)
  • <article> — self-contained content (blog post, news article)
  • <section> — thematic grouping of content with a heading
  • <aside> — tangentially related content (sidebar)
  • <footer> — footer for a page or section

Why it matters: Screen readers, search engines, and developers all benefit. Google gives more weight to content inside semantic tags.

What is the difference between id and class?

id must be unique on the page — only one element can have a given id. Used for JavaScript targeting and anchor links (#section-id). Higher CSS specificity (1,0,0).

class can be reused on multiple elements. Used for CSS styling and grouping. Lower specificity (0,1,0). Use class for styles, id for JS hooks.

What is accessibility (a11y) and how do you implement it?

Accessibility means making your website usable by people with disabilities — visual, motor, auditory, cognitive.

  • Use semantic HTML — screen readers understand headings, lists, nav
  • Add alt text to all images
  • Ensure keyboard navigation works (tab order, focus states)
  • Use sufficient color contrast (4.5:1 ratio for normal text)
  • Use aria-label for icons with no text
  • Use <label> on form inputs (connect with for and id)
  • Don't rely on color alone to convey meaning
What does defer and async do on a script tag?

Without either, the browser stops parsing HTML to download and execute the script — blocking the page.

  • defer — download in parallel, execute AFTER HTML is fully parsed. Order is maintained. Best for most scripts.
  • async — download in parallel, execute as soon as downloaded (could interrupt HTML parsing). Order not guaranteed. Best for independent scripts like analytics.
<script src="app.js" defer></script>

What is the DOM?

The DOM (Document Object Model) is the browser's in-memory tree representation of your HTML. When the browser parses HTML, it builds a tree of node objects — every tag becomes a node. JavaScript interacts with the DOM to read, add, modify, or remove elements — this is what makes pages dynamic without reloading.

CSS Questions

Layout, specificity, the tricky stuff

8 Questions
What is the CSS Box Model and how does box-sizing: border-box change it?

Every HTML element is a rectangular box with 4 layers: content → padding → border → margin.

By default (content-box): width/height only includes the content area. Adding padding and border makes the element LARGER than specified.

With border-box: width/height includes content + padding + border. The element stays the exact size you set — padding and border carve into it instead of expanding it. This is why every project starts with * { box-sizing: border-box; }.

Explain CSS Specificity. How do you calculate it?

Specificity determines which CSS rule wins when multiple rules target the same element. Calculated as a 3-part score (a, b, c):

  • a = ID selectors (#id) = 1,0,0
  • b = Class, pseudo-class, attribute selectors = 0,1,0
  • c = Type selectors (div, p) = 0,0,1

Example: #nav .item:hover = 1+1+1 = 1,2,1. The highest specificity wins. Inline styles beat all. !important beats inline styles (use only as last resort).

What is the difference between Flexbox and CSS Grid? When do you use each?

Flexbox is one-dimensional — it works in one direction (row OR column). Use it for: navbars, centering things, distributing items in a single line, card layouts where items should wrap naturally.

Grid is two-dimensional — rows AND columns simultaneously. Use it for: page layouts (header/sidebar/main/footer), image galleries, any design where you need precise control over both axes.

Rule of thumb: Component-level layout → Flexbox. Page-level layout → Grid. They also work great together — a Grid layout with Flexbox inside each grid item.

What is the difference between position: relative, absolute, fixed, and sticky?
  • relative — stays in the normal document flow. You can offset it with top/left but it still takes up its original space. Used to create a positioning context for absolute children.
  • absolute — removed from normal flow. Positioned relative to the nearest ancestor with position: relative (or the viewport if none).
  • fixed — removed from flow. Always positioned relative to the viewport. Doesn't move when you scroll. Good for: headers, modals, chat buttons.
  • sticky — acts like relative until it hits a scroll threshold, then acts like fixed. Good for sticky headers that only stick after you scroll past them.
How do you center an element horizontally and vertically in CSS? 
/* Method 1: Flexbox (most common) */
.parent {
  display: flex;
  justify-content: center;
  align-items: center;
}

/* Method 2: Grid */
.parent {
  display: grid;
  place-items: center;
}

/* Method 3: Absolute + transform */
.child {
  position: absolute;
  top: 50%; left: 50%;
  transform: translate(-50%, -50%);
}

/* For text only */
.text { text-align: center; line-height: 100px; }
What is a CSS variable (custom property) and why use them?

CSS variables store values you reuse across your stylesheet. They cascade and can be changed with JavaScript.

:root {
  --color-primary: #e94560;
  --spacing-md: 16px;
}

.btn { background: var(--color-primary); padding: var(--spacing-md); }

/* Dark mode — just override the variables */
@media (prefers-color-scheme: dark) {
  :root { --color-bg: #0f0f1a; }
}

Benefits: Change one value, updates everywhere. Enables themes. Can be updated with JavaScript at runtime.

What is the difference between em, rem, %, and px?
  • px — absolute pixels. Fixed. Doesn't scale with user font preferences.
  • rem — relative to the root (html) font-size (default 16px). 1rem = 16px. Scales with user browser settings. Best for font sizes and spacing.
  • em — relative to the parent's font-size. Compounds in nested elements — can be confusing. Best for padding/margin that should scale with the element's own font size.
  • % — relative to the parent's dimension. Best for widths that should scale with the container.

Best practice: Use rem for typography and spacing. Use % for widths. Avoid em for font sizes to prevent compounding issues.

What is a media query and how do you write mobile-first CSS?

A media query applies CSS rules only when certain conditions are met (screen size, orientation, color scheme).

Mobile-first means writing base styles for mobile, then using min-width queries to add/override for larger screens. This is better than desktop-first because mobile is now the majority of traffic and it produces simpler, cleaner CSS.

/* Base = mobile styles */
.grid { grid-template-columns: 1fr; }

/* Tablet and up */
@media (min-width: 768px) {
  .grid { grid-template-columns: 1fr 1fr; }
}

/* Desktop and up */
@media (min-width: 1024px) {
  .grid { grid-template-columns: repeat(3, 1fr); }
}

JavaScript Questions

The core language — asked in every interview

10 Questions
What is the difference between var, let, and const?
  • var — function-scoped, hoisted, can be re-declared. Avoid it — causes confusing bugs.
  • let — block-scoped, not re-declarable, value can be reassigned. Use when value changes.
  • const — block-scoped, not re-declarable, not re-assignable. Use by default for everything.

Note: const on an object/array doesn't make the contents immutable — you can still push to arrays or modify object properties. It only prevents reassigning the variable itself.

What is the difference between == and ===?

=== (strict equality) checks both value AND type. Always use this.

== (loose equality) only checks value after performing type coercion — converts types automatically. This causes unintuitive behavior:

0 == false    // true  (dangerous!)
"" == false   // true  (dangerous!)
null == undefined // true
5 == "5"      // true

5 === "5"     // false (correct!)

Only acceptable use of ==: value == null to check for both null and undefined at once.

What is a closure?

A closure is when a function remembers and has access to variables from its outer scope even after the outer function has finished running.

const makeCounter = () => {
  let count = 0;  // this variable is "closed over"
  return {
    increment: () => ++count,
    getCount:  () => count
  };
};

const counter = makeCounter();
counter.increment(); // 1
counter.increment(); // 2
counter.getCount();  // 2
// count is still accessible via closure!

Real uses: Private variables, factory functions, event handlers that remember state, the module pattern.

What is the difference between null and undefined?
  • undefined — a variable has been declared but not assigned a value yet. Also returned by functions that have no return statement, and by accessing a non-existent property.
  • null — intentionally set to "no value". A developer explicitly assigned null to signal emptiness.
let x;          // undefined (declared, not assigned)
let y = null;   // null (intentionally empty)

typeof undefined // "undefined"
typeof null      // "object" (famous JS bug)
What is event bubbling and event delegation?

Event bubbling: When an event fires on an element, it also fires on all its ancestors up to the document. Clicking a button also triggers click on its div, section, body, and document.

Event delegation: Instead of adding event listeners to every child element, add one listener to the parent and check event.target to identify which child was clicked. This is better for performance and works for dynamically added elements.

// Event delegation — one listener handles all buttons
list.addEventListener("click", (e) => {
  if (e.target.classList.contains("delete-btn")) {
    e.target.closest("li").remove();
  }
});
What is a Promise? What is async/await?

A Promise is an object representing a future value — either resolved (success) or rejected (failure). It avoids "callback hell" for async operations.

async/await is syntactic sugar over Promises that makes async code look synchronous — easier to read and reason about.

// Promise (older style)
fetch(url)
  .then(res => res.json())
  .then(data => console.log(data))
  .catch(err => console.error(err));

// async/await (modern — use this)
const getData = async () => {
  try {
    const res  = await fetch(url);
    const data = await res.json();
    return data;
  } catch (err) {
    console.error(err);
  }
};
What is the difference between map(), filter(), and reduce()?
  • map() — transforms every item, returns a new array of the SAME length.
  • filter() — keeps only items that pass the test, returns a new array that is SAME or SHORTER.
  • reduce() — collapses an array into a SINGLE value (number, object, string, array).
const nums = [1, 2, 3, 4, 5];

nums.map(n => n * 2);          // [2,4,6,8,10]
nums.filter(n => n > 3);       // [4,5]
nums.reduce((sum,n) => sum+n, 0); // 15

None of them mutate the original array — they all return new arrays/values.

What is the difference between forEach() and map()?
  • forEach() — iterates over an array. Returns undefined. Used when you want to DO something with each item (logging, DOM updates) but don't need the results as an array.
  • map() — transforms each item and returns a NEW array of results. Use when you need the transformed data.
// forEach — side effects, no return value
users.forEach(u => console.log(u.name));

// map — transforms and returns new array
const names = users.map(u => u.name); // ["Alice", "Bob"]
What is hoisting in JavaScript?

Hoisting is JavaScript's behavior of moving declarations to the top of their scope before execution. Function declarations are fully hoisted (you can call them before they're defined). var declarations are hoisted but not their values (value is undefined until assignment). let and const are NOT hoisted in the same way — they're in the "temporal dead zone" until their line.

// Function declaration — works
greet(); // "Hello!" — hoisted
function greet() { console.log("Hello!"); }

// var — hoisted but undefined
console.log(x); // undefined (not error)
var x = 5;

// let/const — temporal dead zone
console.log(y); // ReferenceError
let y = 5;
What is this in JavaScript?

this refers to the object that is executing the current function. Its value depends on HOW the function is called.

  • In a regular function: this = the calling object (or window globally)
  • In an arrow function: this = inherited from the surrounding scope (lexical this)
  • In an event handler: this = the element that received the event
  • In a class method: this = the class instance

Arrow functions don't have their own this — they inherit it from where they're defined. This is why arrow functions are used as React event handlers and class methods.

React Questions

Components, state, hooks — asked at every frontend role

7 Questions
What is the difference between state and props in React?
  • Props — data passed FROM a parent TO a child. Read-only inside the child. Used to configure and customize components. Like function arguments.
  • State — data that lives inside a component and can change over time. When state changes, the component re-renders. Managed with useState.

Rule: If a component only needs to receive and display data → props. If a component needs to track something that changes over time → state. If multiple components need the same data → lift state up to their common parent.

What is the Virtual DOM and why does React use it?

The Virtual DOM is a lightweight JavaScript copy of the real DOM that React keeps in memory. When state changes, React first updates the Virtual DOM, then uses a "diffing" algorithm to find what actually changed, and then only updates those specific parts of the real DOM (this is called reconciliation).

Why? Real DOM updates are expensive. By batching and minimizing DOM operations, React makes complex UIs much faster than manually manipulating the DOM.

What are React hooks? Name the most important ones.

Hooks are functions that let you "hook into" React features from function components. You can't use them inside classes or regular functions — only React components and custom hooks.

  • useState — add state to a component
  • useEffect — run side effects (fetch, timers, subscriptions)
  • useRef — store a mutable value that doesn't cause re-render, or access DOM directly
  • useContext — read from a React Context (shared global state)
  • useMemo — memoize expensive calculations
  • useCallback — memoize functions
What is the purpose of the key prop in React lists?

The key prop helps React identify which items in a list have changed, been added, or been removed. React uses keys to perform efficient DOM updates during reconciliation — without keys, it would re-render the entire list on every change.

Rules:

  • Keys must be unique among siblings (not globally)
  • Use stable IDs from your data, not array indexes
  • Using array index as key causes bugs when items are reordered or deleted
What is the useEffect dependency array and what happens with each option? 
// No array — runs after EVERY render
useEffect(() => { ... });

// Empty array — runs ONCE on mount only
useEffect(() => { ... }, []);

// With values — runs when any value changes
useEffect(() => { ... }, [userId, search]);

// With cleanup — runs on unmount / before next effect
useEffect(() => {
  const timer = setInterval(tick, 1000);
  return () => clearInterval(timer); // cleanup
}, []);

Rule: Every value you use inside useEffect that comes from outside it (props, state, variables) must be in the dependency array. ESLint's exhaustive-deps rule enforces this.

What is prop drilling and how do you solve it?

Prop drilling is when you have to pass props through many layers of components just to get data to a deeply nested component that needs it — even if intermediate components don't use it.

Solutions:

  • React Context — built-in solution for global/shared state (theme, auth user, language)
  • State management library — Zustand (simple) or Redux (complex apps)
  • Component composition — restructure components so children is passed as props instead of drilling
What is the difference between controlled and uncontrolled components in React?

Controlled — the form input's value is controlled by React state. Every change updates state via onChange. React is the single source of truth. You can validate and transform input in real time.

Uncontrolled — the DOM manages the input's value. You read it with a ref when you need it. Simpler for basic forms but less powerful.

// Controlled (recommended)
const [name, setName] = useState("");
<input value={name} onChange={e => setName(e.target.value)} />

// Uncontrolled
const inputRef = useRef();
<input ref={inputRef} />
// Read with: inputRef.current.value

General / Behavioral

How you think, work, and communicate

5 Questions
How do you debug a CSS layout issue?
  1. Open browser DevTools (F12) → Elements tab
  2. Click the element that's wrong
  3. Check the Computed tab for actual width/height/margin values
  4. Add outline: 2px solid red to see the box boundaries
  5. Check if a parent has overflow: hidden clipping the child
  6. Check if z-index issue — inspect stacking context
  7. Try * { outline: 1px solid red } to visualize all boxes
How do you make a website perform better?
  • Optimize images — compress, use WebP format, add loading="lazy"
  • Minify CSS and JavaScript (build tools do this automatically)
  • Use a CDN for static assets
  • Reduce HTTP requests — combine files where possible
  • Use defer on script tags
  • Avoid layout-triggering CSS in animations (use transform and opacity instead)
  • Preload critical fonts with <link rel="preload">
  • In React — lazy load routes, memoize expensive components
Explain the difference between GET and POST HTTP methods.
  • GET — retrieve data. Parameters go in the URL. Can be cached, bookmarked, shared. No body. Used for reading.
  • POST — send data to the server. Data goes in the request body. Not cached. Used for creating, logging in, submitting forms.

Other methods: PUT (replace resource), PATCH (partial update), DELETE (remove resource). These form the foundation of REST APIs.

What is version control and how do you use Git in your workflow?

Git tracks changes to your code over time, lets you revert mistakes, and enables collaboration. My basic workflow:

  • git init — start tracking a project
  • git add . — stage changes
  • git commit -m "message" — save a snapshot
  • git branch feature-name — new branch for each feature
  • git push origin main — push to GitHub
  • Open a Pull Request → review → merge to main

Never commit directly to main. Each feature gets its own branch.

Tell me about a project you built and what you learned from it.

Structure your answer with STAR: Situation → Task → Action → Result.

Example: "I built a weather app that fetches real-time data from the OpenWeatherMap API. I ran into a challenge with async data loading — the component would render before the data arrived, causing a crash. I solved this by adding loading and error states and only rendering the data when it was available. I learned that every async operation needs three states: loading, success, and error — and that's now a pattern I use in every project."

Key points to hit: What problem it solved, a challenge you faced, how you solved it, what you learned. Be specific — say exact technologies.

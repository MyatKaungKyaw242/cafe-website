HTML & General Web
HTML HTML
HyperText Markup Language. The structure of a web page — it defines what content exists (headings, paragraphs, images, buttons) but not how it looks. Like a skeleton — no skin or muscles, just the bones.
Element HTML
A single piece of HTML — the opening tag, content, and closing tag together. <p>Hello</p> is one element. Everything you see on a page is made of elements.
Attribute HTML
Extra information added to an HTML tag. src, href, class, id are all attributes. Written inside the opening tag. <img src="photo.jpg" alt="A cat"> — src and alt are attributes.
DOM General
Document Object Model. When a browser loads your HTML, it converts it into a tree of objects that JavaScript can read and change. The DOM is that live tree. document.getElementById("btn") finds a node in the DOM tree.
Semantic HTML HTML
Using HTML tags that describe their meaning, not just their appearance. <nav>, <article>, <footer> are semantic. <div> and <span> have no meaning. Screen readers and search engines use semantic tags to understand your page.
URL / Endpoint General
A URL is a web address. An endpoint is a specific URL on a server that accepts requests and returns data. https://api.github.com/users/octocat is an endpoint that returns a user's data.
HTTP / HTTPS General
HTTP is the protocol (set of rules) for sending data between your browser and a server. HTTPS is the secure, encrypted version. Always use HTTPS for real sites. When you visit a site, your browser sends an HTTP request; the server sends back a response.
Request & Response General
A request is your browser asking a server for something. A response is what the server sends back. Every page load and every API call is a request-response cycle. You click a link → browser sends a request → server sends back HTML → browser renders it.
API General
Application Programming Interface. A way for two programs to talk to each other. A web API is a URL you can call to get or send data — usually JSON. Weather app fetches from a weather API to get today's temperature.
JSON General
JavaScript Object Notation. A text format for storing and sending data. It looks like a JS object. APIs almost always return JSON. { "name": "Myat", "age": 20 } — that's JSON.
localhost General
Your own computer acting as a server. When you run npm run dev, your site is available at localhost:5173 — only on your machine. It's like a test environment before you publish to the real internet.
Deployment / Hosting General
Putting your website on the internet so anyone can access it. Services like Netlify, Vercel, and GitHub Pages host your files for free. You build the site locally → deploy to Netlify → now anyone in the world can visit it.
CSS
CSS CSS
Cascading Style Sheets. Controls how HTML elements look — color, size, font, layout, spacing, animations. If HTML is the skeleton, CSS is the skin and clothes.
Selector CSS
The part of a CSS rule that targets which HTML elements to style. .card targets elements with class "card". #header targets an element with id "header". .card { background: blue; } — .card is the selector.
Box Model CSS
Every HTML element is a box with 4 layers: content → padding → border → margin. Understanding this explains why things don't line up the way you expect. Padding is space inside the element. Margin is space outside it.
Specificity CSS
When two CSS rules target the same element, specificity decides which one wins. ID > class > tag. More specific = wins. #btn beats .btn beats button.
Cascade CSS
The order CSS rules are applied. If two rules have equal specificity, the one that comes last in the file wins. The "C" in CSS — styles flow down and the last one wins ties.
Flexbox CSS
A CSS layout mode for arranging items in a row or column. The parent controls how children are aligned and spaced using display: flex. justify-content: center centers all children horizontally.
CSS Grid CSS
A 2D layout system — you define rows AND columns, then place items anywhere in the grid. Better than Flexbox for full page layouts. grid-template-columns: repeat(3, 1fr) creates 3 equal columns.
Responsive Design CSS
Building layouts that adapt to different screen sizes — mobile, tablet, desktop. Done with media queries and flexible units like %, vw, rem. The same site looks good on a phone and on a TV.
Media Query CSS
A CSS rule that applies styles only when a condition is true — like the screen being smaller than 600px. @media (max-width: 600px) { /* mobile styles */ }
CSS Variables CSS
Custom properties that store reusable values. Defined with --name and used with var(--name). Change once, updates everywhere. :root { --accent: #61dafb; } then use color: var(--accent).
rem / em / px CSS
px = fixed pixels. rem = relative to the root font size (usually 16px). em = relative to the parent's font size. Use rem for font sizes and spacing. 1rem = 16px. 1.5rem = 24px.
Pseudo-class CSS
A keyword added to a selector that styles the element in a specific state. :hover, :focus, :first-child are pseudo-classes. button:hover { background: blue; } — styles on hover.
JavaScript
Variable JS
A named container that stores a value. const can't be reassigned. let can. Avoid var. const name = "Myat"; — name holds "Myat".
Function JS
A reusable block of code that does a specific task. You define it once and call it as many times as you need. const greet = (name) => `Hello, ${name}!`;
Arrow Function JS
A shorter way to write a function using =>. Behaves the same as a regular function in most cases but has different this binding. const add = (a, b) => a + b;
Array JS
An ordered list of values. Can hold any type — numbers, strings, objects, other arrays. const fruits = ["apple", "mango", "banana"];
Object JS
A collection of key-value pairs. Used to group related data and behaviour together. const user = { name: "Myat", age: 20, role: "admin" };
Callback JS
A function passed to another function as an argument, to be called later. Most common in event listeners and array methods. btn.addEventListener("click", handleClick) — handleClick is a callback.
Promise JS
An object that represents a future value — something that will be available later (like API data). Has three states: pending, fulfilled, rejected. fetch(url) returns a Promise.
async / await JS
async marks a function as asynchronous. await pauses execution until a Promise resolves. Makes async code read like synchronous code. const data = await fetch(url); waits for the response.
Destructuring JS
A shorthand to extract values from objects or arrays into named variables. const { name, age } = user; — pulls name and age out of user.
Spread Operator JS
... spreads the items of an array or object into a new one. Used to copy and add without mutating the original. const newArr = [...oldArr, newItem];
Scope JS
Where a variable can be accessed. const and let are block-scoped — they only exist inside the {} where they were declared. A variable declared inside an if block can't be used outside it.
Closure JS
A function that remembers variables from the scope where it was created, even after that scope has finished running. A counter function that "remembers" its count between calls uses a closure.
Type Coercion JS
JavaScript automatically converting one type to another. "5" + 1 gives "51" (string) not 6. Use === instead of == to avoid it. 0 == false is true (coercion). 0 === false is false (strict).
null vs undefined JS
undefined = a variable exists but has no value yet. null = intentionally empty. Both are "nothing" but come from different sources. let x; → x is undefined. let y = null; → y is deliberately empty.
Event Listener JS
Code that waits for something to happen (a click, keypress, scroll) and runs a function when it does. btn.addEventListener("click", fn) runs fn every time the button is clicked.
localStorage JS
Browser storage that saves data as strings and persists even after you close the browser tab. Use JSON to store objects. localStorage.setItem("theme", "dark") saves theme setting.
React
Component React
A reusable piece of UI — a JavaScript function that returns JSX. React apps are built by combining components like LEGO bricks. Navbar, Button, UserCard are all components.
JSX React
JavaScript XML. Looks like HTML written inside JavaScript. React compiles it to regular JS function calls. It's not real HTML — it's syntactic sugar. return <h1>Hello</h1>; — that's JSX inside a JS function.
Props React
Short for properties. Values passed from a parent component to a child component — like HTML attributes but for your own components. Props are read-only. <Button label="Submit" onClick={handleSubmit} />
State React
Data that belongs to a component and can change over time. When state changes, React re-renders the component to show the new data. A counter's current number is state. The user's input in a text field is state.
Hook React
A function that starts with "use" and lets you add React features (like state or effects) to function components. useState, useEffect, useRef are all built-in hooks.
useState React
The hook for declaring reactive state in a component. Returns a value and a setter function. Calling the setter re-renders the component. const [count, setCount] = useState(0);
useEffect React
The hook for running side effects after a render — fetching data, setting up timers, subscribing to events. Takes a function and a dependency array. useEffect(() => { fetchData(); }, []); — runs once on mount.
Re-render React
When React calls your component function again to produce updated JSX. Triggered by state changes or prop changes. React updates only what changed in the DOM. Clicking a button that calls setCount triggers a re-render.
Virtual DOM React
A lightweight copy of the real DOM that React keeps in memory. When state changes, React compares the new virtual DOM to the old one and only updates the real DOM where things changed. This is why React is fast — it avoids rebuilding the whole page on every change.
Lifting State Up React
Moving state from a child component up to the parent so sibling components can share it. The parent owns the state and passes it down as props. Two siblings that need the same data → put the state in their parent.
Controlled Component React
A form input whose value is controlled by React state — not the DOM. The input's value always reflects the state, and every change updates the state. <input value={name} onChange={e => setName(e.target.value)} />
Custom Hook React
A function starting with "use" that you write yourself to extract and reuse stateful logic across components. useFetch, useLocalStorage, useDebounce are common custom hooks.
Prop Drilling React
Passing a prop through many layers of components just to get it to a deeply nested child. Sign that you should use React Context instead. App → Page → Section → Card → Button — all just to pass a username.
Context React
A way to share data globally across components without prop drilling. Create a context, provide it at the top, consume it anywhere below. Theme, language, and auth state are typically stored in Context.
Key Prop React
A special prop you add to elements in a list so React can identify which item changed. Must be unique and stable — use the item's ID, not the index. <Card key={item.id} /> inside a .map().
Side Effect React
Anything that affects something outside the component — fetching data, updating the document title, setting a timer, writing to localStorage. Side effects go in useEffect, not directly in the component body.

HTML & CSS Tips & Tricks

A reference guide — open in Chrome and press Ctrl+P to save as PDF

CSS Tips & Tricks

1. box-sizing: border-box

Padding and border are included inside the width. Without this, layouts break unexpectedly.

* {
    box-sizing: border-box; /* always put this in your reset */
}

.box {
    width: 300px;
    padding: 20px; /* stays inside 300px, doesn't add on top */
}

2. display: block on img

Images are inline by default, which creates a small gap underneath. display: block removes it.

img {
    display: block;  /* removes bottom gap caused by text baseline */
    max-width: 100%; /* never overflows its container */
}

3. max-width: 100% on img

Prevents images from overflowing their container on small screens.

img {
    max-width: 100%; /* shrinks to fit any container */
}

4. margin: 0 auto

Centers a block element horizontally. Requires a defined width.

.card {
    width: 400px;
    margin: 0 auto; /* 0 = top/bottom, auto = left/right */
}
Only works on block elements with a defined width.

5. min-height: 100vh

Makes the page always fill the full screen height. Use min-height so content can still expand.

body {
    min-height: 100vh; /* full screen height, expands if content is taller */
}

6. object-fit: cover

Fills the container without stretching the image. Like background-size: cover but for img tags.

img {
    width: 300px;
    height: 200px;
    object-fit: cover;       /* fills box, crops excess */
    object-position: center; /* crops from center */
}

7. gap in flex/grid

Cleaner than using margin on children. Works in both flexbox and grid.

.container {
    display: flex;
    gap: 16px; /* spacing between ALL children, no extra fixes needed */
}

8. position: relative + position: absolute

The child positions itself relative to the nearest parent with position: relative.

.card {
    position: relative; /* sets the boundary for children */
}

.badge {
    position: absolute; /* positions inside .card */
    top: 10px;
    right: 10px;
}
Always ask: does the parent have position: relative?

9. z-index only works on positioned elements

z-index is ignored on static elements. You need position set first.

/* Does NOTHING */
.box { z-index: 999; }

/* Works */
.box {
    position: relative; /* or absolute, fixed, sticky */
    z-index: 999;
}

10. overflow: hidden

Clips children that go outside the container. Also useful for zoom-on-hover effects.

.card {
    overflow: hidden;
    border-radius: 10px;
}

.card img {
    transition: transform 0.3s ease;
}

.card:hover img {
    transform: scale(1.1); /* zoom stays clipped inside card */
}

11. clamp(min, preferred, max)

Responsive sizing without media queries.

h1 {
    font-size: clamp(1.5rem, 4vw, 3rem);
    /* small screen = 1.5rem, scales with viewport, max 3rem */
}

12. CSS Variables

Define values once in :root, reuse everywhere. Change in one place, updates everywhere.

:root {
    --primary: #ff6600;
    --radius: 8px;
}

button {
    background: var(--primary);
    border-radius: var(--radius);
}

a {
    color: var(--primary);
}

13. transition

Smooth animations on hover without JavaScript.

button {
    background: blue;
    transition: background 0.3s ease, transform 0.2s ease;
}

button:hover {
    background: darkblue;
    transform: scale(1.05);
}
Format: transition: property duration timing-function

14. :root for global settings

:root is the html element — the highest level. Variables here are available everywhere in CSS.

:root {
    --color-bg: #1a1a2e;
    --color-text: #eaeaea;
    --spacing-lg: 2rem;
}

body {
    background: var(--color-bg);
    color: var(--color-text);
}

15. text-overflow: ellipsis

Cuts long text with ... instead of overflowing. All 3 lines required together.

.title {
    white-space: nowrap;     /* step 1: prevent wrapping */
    overflow: hidden;         /* step 2: hide overflow */
    text-overflow: ellipsis; /* step 3: show ... */
    width: 200px;
}
/* Result: "This is a very long title th..." */

HTML Tips & Tricks

1. lang="en" on html

Tells the browser and screen readers what language the page is in. Affects SEO and accessibility.

<html lang="en">

2. alt text on img

Empty alt for decorative images, descriptive text for content images. Google reads alt for SEO.

<!-- Decorative — screen reader skips it -->
<img src="divider.png" alt="">

<!-- Content — describe what it shows -->
<img src="profile.jpg" alt="John Doe smiling in a blue shirt">

3. Semantic tags

Tell the browser what the content means. Better for SEO, accessibility, and readability.

<!-- Bad -->
<div class="header">...</div>
<div class="nav">...</div>

<!-- Good -->
<header>...</header>
<nav>...</nav>
<main>...</main>
<footer>...</footer>

4. meta viewport

Without this, mobile browsers zoom out and show a desktop page. Media queries won't work.

<meta name="viewport" content="width=device-width, initial-scale=1.0">

5. label for linked to inputs

Clicking the label focuses the input. Better UX and accessibility.

<!-- Bad — label does nothing on click -->
<label>Email</label>
<input type="email" id="email">

<!-- Good — clicking "Email" focuses the input -->
<label for="email">Email</label>
<input type="email" id="email">

6. loading="lazy" on img

Images only load when scrolled into view. Makes page load faster.

<img src="hero.jpg" alt="Hero" loading="eager">  <!-- loads immediately -->
<img src="photo.jpg" alt="..." loading="lazy">   <!-- loads when visible -->

7. rel="noopener noreferrer" on external links

Without rel, the new tab can access your page via window.opener — a security vulnerability.

<a href="https://google.com" target="_blank" rel="noopener noreferrer">
    Visit Google
</a>
Always add rel when using target="_blank".

8. placeholder vs value

placeholder is the grey hint text. value is the actual pre-filled value. They do different things.

<!-- placeholder — hint text, disappears when user types -->
<input type="text" placeholder="Enter your name">

<!-- value — pre-filled actual value -->
<input type="text" value="John Doe">

9. button type

Inside a form, buttons default to type="submit" which reloads the page. Always specify the type.

<button type="submit">Submit Form</button>
<button type="button">Just a Button</button>
<button type="reset">Reset Form</button>

10. fieldset for grouping form inputs

Groups related fields visually and semantically. Screen readers announce the legend when entering the group.

<form>
    <fieldset>
        <legend>Personal Info</legend>
        <label for="name">Name</label>
        <input type="text" id="name">
    </fieldset>

    <fieldset>
        <legend>Address</legend>
        <label for="city">City</label>
        <input type="text" id="city">
    </fieldset>
</form>
